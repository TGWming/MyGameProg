## 优化 1：螺旋楼梯实际测试

### 当前状态
- 理论上支持，逐段宽度计算已考虑方向旋转
- 未经实际测试

### 测试计划
```
1. 找一个螺旋楼梯 Mesh（面数较高的）
2. 生成 Spline → 手动铺设 8-12 个控制点
3. 每段弧度控制在 90° 以内
4. 生成 Ramp → 检查：
   a. 每段宽度是否合理？
   b. 碰撞体是否覆盖楼梯踏面？
   c. 行走时是否顺滑？
   d. 是否有缝隙或重叠？
```

### 可能的问题

```
俯视图 — 螺旋楼梯顶点筛选问题：

        P3
       /      ← 上层
      /   
  P4 ·     · P2 ← 中层
     |     |
     |     |
  P5 ·     · P1 ← 下层
      \   /
       \ /
        P0

问题：相邻但不同高度的段，顶点在 XY 平面上重叠
当前方案：用 Z 高度范围筛选（± 50 容差）
可能需要：减小容差 或 增加角度范围筛选
```

### 潜在改进
- 如果矩形 Box 不贴合扇形踏面，考虑梯形碰撞体
- 可能需要增加 Spline 采样密度（用 Distance 而非 Point）

## 优化 2：Ramp Thickness 自动匹配台阶高度

### 当前状态
- Thickness 固定为 5.0（用户手动设置）
- 太薄：角色可能穿过
- 太厚：角色走起来像踩在空气上

### 实现思路

```
侧视图 — 楼梯台阶：

  ┌──┐
  │  ├──┐           Ramp 斜面（虚线）
  │  │  ├──┐      ╱
  │  │  │  ├──┐ ╱   ← Thickness 应匹配台阶凹凸深度
  │  │  │  │  │╱
  └──┴──┴──┴──┘

  台阶高度 StepHeight = 单级台阶的高
  Thickness ≈ StepHeight / 2（经验值）
```

### 方案 A：从 Mesh 顶点分析台阶高度
```
1. 提取 Mesh 顶点的 Z 值
2. 对 Z 值排序 → 找到离散的高度层
3. 相邻层的 Z 差值 = 单级台阶高度
4. Thickness = StepHeight * 0.4 ~ 0.6
```

### 方案 B：用户输入
```
在 CreateStairSpline 的 Tag 中存储：
  "StepHeight_XX.X"
用户在 Widget 中输入台阶高度
或者从 Mesh 自动检测后存储
```

## 优化 3：一键清除旧 Ramp 的按钮

### 当前状态
- 每次重新生成 Ramp 需要手动删除旧的
- 多次调试时很繁琐

### 实现思路

```
新增函数：DeleteStairRamps

1. 遍历关卡中所有 Actor
2. 查找 ActorLabel 包含 "StairRamp" 或 "StairSpline" 的 Actor
3. 或者查找 Tags 中包含 "StairWidth_" 的 Actor
4. 调用 DestroyActor()

蓝图连线：
  Button_2 OnClicked → DeleteStairRamps(Self)
```

### 高级版本
```
选中一个 Spline → 只删除该 Spline 对应的 Ramp
（需要在 Ramp Actor 的 Tag 中存储对应 Spline 的名字）
```

## 优化 4：Spline 控制点自动沿楼梯路径生成

### 当前状态
- Spline 只生成 2 个端点（底部 + 顶部）
- L 形 / U 形 / 螺旋需要手动加点和调整

### 实现思路 — 基于 Mesh 顶点聚类

```
原理：
  1. 提取楼梯 Mesh 所有顶点
  2. 按 Z 高度分层（每级台阶一层）
  3. 每层计算 XY 质心
  4. 质心连线 = 楼梯路径
  5. 路径点简化（Douglas-Peucker 算法）→ Spline 控制点

俯视图 — 自动检测 L 形路径：

  · · · ·  Z层8     ← 质心 = 上层中心
  · · · ·  Z层7
  · · · · · · · ·  Z层5-6  ← 质心 = 平台中心
  · · · ·  Z层3-4
  · · · ·  Z层1-2   ← 质心 = 下层中心

  自动生成 3 个控制点：底部、平台、顶部 ✅
```

### 复杂度评估
```
难度：★★★★☆（中高）
原因：
  - Z 分层需要自适应阈值（不同楼梯台阶高度不同）
  - 质心计算在 L 形拐弯处可能偏移
  - 螺旋楼梯的质心可能在楼梯外侧
  - 需要大量测试不同楼梯 Mesh

建议：作为后期高优先级优化，当前手动调整已可接受
```
