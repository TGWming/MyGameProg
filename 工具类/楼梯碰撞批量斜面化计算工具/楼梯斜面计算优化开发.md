## 二、优化 A：Ramp Thickness 自动匹配台阶高度

### 2.1 问题描述

```
侧视图 — 当前固定 Thickness 的问题：

  Thickness = 5（太薄）：         Thickness = 50（太厚）：

  角色 ○                          角色 ○
         ╲  ← 薄纸片                    ╱▓▓▓▓▓╲  ← 厚砖块
  ┌──┐    ╲                      ┌──┐ ╱▓▓▓▓▓▓▓╲
  │  ├──┐  ╲                     │  ├▓▓▓▓▓▓▓▓▓╲
  │  │  ├──┐╲                    │  │  ├▓▓▓▓▓▓▓╲
  └──┴──┴──┘                     └──┴──┴──┘
  
  脚踩在台阶边缘，视觉浮空        脚踩在空气中，离台阶面太远
```

### 2.2 理想效果

```
侧视图 — 自动匹配后：

  角色 ○
         ╲▓▓╲   ← Thickness ≈ 台阶高度 × 0.5
  ┌──┐    ╲▓▓╲      刚好填满台阶凹凸的空隙
  │  ├──┐  ╲▓▓╲     角色脚部看起来踩在台阶面上
  │  │  ├──┐╲▓▓╲
  │  │  │  ├──╲▓╲
  └──┴──┴──┴──┘

  Thickness = StepHeight × 0.5
  行走手感自然，视觉不浮空
```

### 2.3 实现原理

```
核心：从 Mesh 顶点的 Z 值分布中检测台阶高度

Step 1：提取所有顶点的 Z 值（本地坐标）

  已有代码（逐段宽度计算中的 PositionVertexBuffer），直接复用。

Step 2：Z 值排序 + 聚类

  所有顶点的 Z 值（排序后）：

  Z=0.0   ····  ← 第 1 簇（第 1 级台阶踏面）
  Z=0.5   ··
  Z=1.0   ·
  Z=20.0  ····  ← 第 2 簇（第 2 级台阶踏面）
  Z=20.5  ··
  Z=21.0  ·
  Z=40.0  ····  ← 第 3 簇（第 3 级台阶踏面）
  Z=40.5  ··
  Z=60.0  ····  ← 第 4 簇

  簇内顶点 Z 值很接近（< 5 单位）
  簇间间距 ≈ 20 = 台阶高度

Step 3：计算相邻簇间距

  簇中心：0, 20, 40, 60
  间距：  20, 20, 20
  取中位数（避免异常值影响）→ StepHeight = 20

Step 4：计算 Thickness

  Thickness = StepHeight × 0.5 = 10.0
  Clamp(3.0, 50.0) → 安全范围限制

Step 5：存入 Tag

  SplineActor->Tags.Add("StepHeight_20.0")
  GenerateRampFromSpline 读取 Tag → 使用自动 Thickness
```

### 2.4 算法伪代码

```
// 提取 Z 值
TArray<float> ZValues;
for (FVector& Vert : MeshVertices)
    ZValues.Add(Vert.Z);
ZValues.Sort();

// 聚类：相邻 Z 差 < 5.0 的归为同一簇
TArray<float> ClusterCenters;
float ClusterSum = ZValues[0];
int32 ClusterCount = 1;

for (int32 i = 1; i < ZValues.Num(); i++)
{
    if (ZValues[i] - ZValues[i-1] < 5.0f)
    {
        ClusterSum += ZValues[i];
        ClusterCount++;
    }
    else
    {
        ClusterCenters.Add(ClusterSum / ClusterCount);
        ClusterSum = ZValues[i];
        ClusterCount = 1;
    }
}
ClusterCenters.Add(ClusterSum / ClusterCount); // 最后一簇

// 计算相邻簇间距
TArray<float> StepHeights;
for (int32 i = 1; i < ClusterCenters.Num(); i++)
    StepHeights.Add(ClusterCenters[i] - ClusterCenters[i-1]);

// 取中位数
StepHeights.Sort();
float StepHeight = StepHeights[StepHeights.Num() / 2];

// 最终 Thickness
float AutoThickness = FMath::Clamp(StepHeight * 0.5f, 3.0f, 50.0f);
```

### 2.5 风险评估

| 风险 | 等级 | 说明 | 应对方案 |
|------|------|------|----------|
| 栏杆/装饰物顶点干扰 | ⚠️ 中 | 栏杆顶点的 Z 值会混入台阶 Z 分布 | 预处理：只取 BoundingBox 内 XY 范围 80% 的顶点（排除边缘栏杆） |
| 台阶高度不均匀 | ⚠️ 中 | 某些楼梯底部台阶和顶部台阶高度不同 | 用中位数而非平均值，抗异常值能力强 |
| 平台区域无台阶 | 🟢 低 | L 形平台没有台阶，Z 值聚类间距很大 | 忽略超过 StepHeight × 3 的间距（视为平台，非台阶） |
| 聚类阈值 5.0 不适用 | 🟢 低 | 极小或极大的楼梯可能台阶内顶点 Z 差 > 5 | 可改为自适应阈值：BBox.SizeZ / (预估台阶数 × 10) |

### 3.4 算法伪代码

```
// Step 3: 按 Z 分层
float LayerHeight = StepHeight * 2.0f;
int32 NumLayers = FMath::CeilToInt(BBoxSizeZ / LayerHeight);

TArray<TArray<FVector>> Layers;
Layers.SetNum(NumLayers);

for (FVector& Vert : MeshVertices)
{
    int32 LayerIndex = FMath::Clamp(
        FMath::FloorToInt((Vert.Z - BBoxMinZ) / LayerHeight),
        0, NumLayers - 1);
    Layers[LayerIndex].Add(Vert);
}

// Step 4: 每层质心
TArray<FVector> Centroids;
for (int32 i = 0; i < NumLayers; i++)
{
    if (Layers[i].Num() < 10) continue; // 跳过顶点太少的层
    
    FVector Sum = FVector::ZeroVector;
    for (FVector& V : Layers[i])
        Sum += V;
    Sum /= Layers[i].Num();
    
    // Z 值用该层范围的中心
    Sum.Z = BBoxMinZ + LayerHeight * (i + 0.5f);
    Centroids.Add(Sum);
}

// Step 6: Douglas-Peucker 简化
TArray<FVector> SimplifiedPath = DouglasPeucker(Centroids, Tolerance);

// Step 7: 设置 Spline 控制点
SplineComp->ClearSplinePoints();
for (int32 i = 0; i < SimplifiedPath.Num(); i++)
{
    FVector LocalPoint = SimplifiedPath[i] - ActorLocation;
    SplineComp->AddSplinePoint(LocalPoint, ESplineCoordinateSpace::Local);
}
```

### 3.5 风险评估

| 风险 | 等级 | 说明 | 应对方案 |
|------|------|------|----------|
| **L 形拐弯质心偏移** | 🔴 高 | 平台是正方形，质心在中心而非拐弯角 | 检测方向突变 → 修正质心到方向变化的角点 |
| **螺旋楼梯中空质心** | 🔴 高 | 螺旋中心是空的，质心落在空洞中 | 使用加权质心：排除距离中心 < R_inner 的顶点 |
| **栏杆顶点干扰质心** | ⚠️ 中 | 栏杆偏向一侧，质心被拉偏 | 预处理：只取踏面高度附近的顶点（Z 值在簇中心 ± StepHeight×0.3） |
| **Z 分层阈值不适配** | ⚠️ 中 | 不同楼梯台阶高度差异大 | 依赖优化 A 的自动 StepHeight 检测结果 |
| **Douglas-Peucker 阈值** | ⚠️ 中 | 阈值太大删掉拐弯点，太小保留太多点 | 默认阈值 = StairWidth × 0.3，可调参 |
| **多 Mesh 组合楼梯** | 🟢 低 | 楼梯由多个 Mesh Actor 拼接 | 当前方案只处理单个 Actor，多 Actor 暂不支持 |

### 3.6 L 形拐弯质心偏移 — 详细分析

```
这是最大的风险点，详细说明：

L 形楼梯平台（俯视图）：
  
  ┌───────────────┐
  │               │
  │               │
  │     ★ 质心    │  ← 质心在正方形中心
  │               │
  │               │
  └───────────────┘
        ↑
      下层入口
                  → 上层出口

  理想控制点应该在：
  
  ┌───────────────┐
  │               │
  │               │
  │               │
  │               │
  │               ★  ← 拐弯的角点（下层到上层的过渡点）
  └───────────────┘

修正算法：
  1. 计算前一层到当前层的方向 Dir_in
  2. 计算当前层到后一层的方向 Dir_out
  3. 如果 Dir_in 和 Dir_out 夹角 > 45° → 这是拐弯层
  4. 拐弯层的控制点 = 质心 + 偏移
     偏移方向 = (Dir_in + Dir_out).Normalize() × 偏移量
     偏移量 = 该层顶点分布的半径 × 0.5

这个修正逻辑是复杂度最高的部分。
```

### 3.7 螺旋楼梯中空质心 — 详细分析

```
螺旋楼梯截面（俯视图）：

      · · · · ·
    · ·       · ·
    ·    空     ·      ← 中间是空心的
    · ·       · ·
      · · · · ·

  简单质心 ★ 落在空心中央 → 错误！

  正确做法：
  
  方案 A：环形质心
    只取距离中心 > R_inner 的顶点
    R_inner = 从所有顶点到 XY 中心的距离的最小值
    
  方案 B：角度分段质心
    按角度将顶点分成扇形区域
    每个区域取自己的质心 → 质心在踏面上而非空洞中

  推荐方案 B，因为它同时解决了"质心在踏面上"的问题。
```
