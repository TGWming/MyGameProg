一、项目概述
================================================================================

我们正在为 Unreal Engine 4.27 开发一个名为 "FlowVisualizer" 的独立编辑器插件。

核心功能：
- 运行时 C++ 数据流可视化工具
- 终端风格（黑底 + 高亮彩色文字 + ASCII 箭头 + 流动光点动画）
- 独立弹出窗口（SWindow），不是编辑器内嵌 Tab
- 实时显示游戏代码的执行链路、模块依赖、数据流动
- 通用插件，适配所有 UE4.27 项目，不绑定任何特定项目代码

使用方式：
- 将 FlowVisualizer 文件夹放入任意 UE4.27 项目的 Plugins/ 目录
- 编辑器内 Edit → Plugins → 搜索 "FlowVisualizer" → 勾选启用 → 重启
- 菜单栏 Window → Flow Visualizer → 弹出独立黑色终端风格窗口

插件结构：
- FlowVisualizerRuntime（运行时模块）：提供 FLOW_SIGNAL 宏和 FFlowTracer 单例
- FlowVisualizerEditor（编辑器模块）：提供独立弹出窗口和终端风格渲染

================================================================================
五、安全约束（每个 Prompt 必须包含）
================================================================================

以下安全约束必须完整写入每一个 Task Prompt 的开头：

1. 作用域锁定：你只允许在以下目录内创建/修改文件：
     <项目根目录>/Plugins/FlowVisualizer/
   禁止触碰任何其他目录，没有例外。

2. 禁止修改项目现有代码：
   - 禁止修改 Source/soul/ 下的任何文件
   - 禁止修改 FlowVisualizer 以外的任何 .Build.cs
   - 禁止修改任何 .uproject 文件
   - 禁止修改任何 Config/ 文件
   - 禁止修改任何 Content/ 资产
   - 禁止修改任何其他插件的文件

3. 禁止向现有模块注入新依赖：
   - 禁止向 soul.Build.cs 添加 "FlowVisualizerRuntime"
   - 禁止向任何现有 .cpp 添加 #include "FlowTracer.h"
   - 该插件必须 100% 自包含

4. 编译隔离：
   - 插件必须能够独立编译
   - 禁用插件后项目必须与之前完全一致
   - 启用插件不得要求修改任何项目代码

5. 文件编码强制规范：
   - 所有新建文件必须使用 UTF-8 编码（无 BOM）
   - 所有代码注释必须使用英文，禁止中文注释，防止编码乱码
   - 与项目现有代码文件的编码格式保持一致

6. 如果你不确定某个文件是否在 Plugins/FlowVisualizer/ 内部，
   立即停止并询问，禁止猜测。

================================================================================
七、执行流程说明
================================================================================

1. 用户在 GitHub Copilot Chat（本界面）中请求生成 Task Prompt
2. Copilot 按 Round 顺序生成对应 Task 的完整 Prompt（文本框格式）
3. 用户一键复制 Prompt
4. 用户粘贴到 VS Code 中的 Copilot Agent（Claude）Chat
5. VS Code Agent 执行编码任务
6. 用户验证结果（编译测试）
7. 验证通过后回到步骤 1 请求下一个 Task

注意：
- 每个 Round 内的 Task 互不依赖，可以按任意顺序执行
- 到达 MVP 检查点时建议暂停全面验证
- 如果某个 Task 执行失败，不影响同 Round 内其他 Task
