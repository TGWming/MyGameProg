【任务目标】
修改 SFlowVisualizerPanel 的 OnFlowEventReceived()，
在接收到节点事件后，自动推导该节点的入边（Incoming Edge）是否应被激活。
推导逻辑：当 TargetNode 收到信号时，如果其 SourceNode 在 200ms 内
也曾收到信号，说明数据确实沿该 Edge 流动，则标记该 Edge 为激活状态。
同时填充 P6.4 中 Tick 的 Edge 动画检查占位。

【具体要求】

1. 需要修改的文件：

   文��� A：Plugins/FlowVisualizer/Source/FlowVisualizerEditor/Public/SFlowVisualizerPanel.h
   文件 B：Plugins/FlowVisualizer/Source/FlowVisualizerEditor/Private/SFlowVisualizerPanel.cpp

2. SFlowVisualizerPanel.h 中应新增：

   Edge 激活状态存储：
   - TMap<TPair<FName, FName>, double> EdgeLastActiveTime
     Key: TPair<FName, FName>(SourceNodeID, TargetNodeID)
     Value: 该 Edge 最后一次被推导为激活的时间戳

   Edge 推导窗口常量：
   - static const double EdgeInferenceWindowSec = 0.2
     （200ms 推导窗口，即 0.2 秒）

3. OnFlowEventReceived() 中应在现有逻辑（P6.1）之后追加 Edge 推导：

   步骤 1 — 获取当前管线数据：
   - 通过 FFlowPipelineRegistry::Get().FindPipeline(CurrentPipelineID)
   - 如果返回 nullptr，跳过 Edge 推导

   步骤 2 — 遍历当前管线所有 Edges：
   - 对每条 FFlowEdge：
     如果 Edge.TargetNodeID == Event.NodeID：
       // 当前事件的节点是该 Edge 的 Target 端
       // 检查 Source 端是否在 200ms 内也曾激活

       const double* SourceActivePtr = NodeLastActiveTime.Find(Edge.SourceNodeID)
       如果 SourceActivePtr 不为 nullptr：
         double SourceElapsed = Event.Timestamp - *SourceActivePtr
         如果 SourceElapsed >= 0 且 SourceElapsed < EdgeInferenceWindowSec：
           // Source 在 200ms 内刚激活过 → 数据确实沿此 Edge 流动
           TPair<FName, FName> EdgeKey(Edge.SourceNodeID, Edge.TargetNodeID)
           EdgeLastActiveTime.Add(EdgeKey, Event.Timestamp)

4. Tick() 中填充 P6.4 预留的 Edge 动画检查占位：

   替换 TODO 注释为实际逻辑：
   - 遍历 EdgeLastActiveTime：
     for (const auto& Pair : EdgeLastActiveTime)
       double Elapsed = CurrentTime - Pair.Value
       如果 Elapsed < FlowVisStyle::HighlightFadeDuration：
         bHasActiveAnimation = true
         break（已确认有活跃动画，无需继续遍历）

5. Clear 按钮逻辑补充：
   - P4.2 中 Clear 按钮已调用 FFlowTracer::Get().ClearBuffer()
   - 本任务应在 Clear 逻辑中追加：
     EdgeLastActiveTime.Empty()
   - 或者留注释说明 P9.1 会统一处理 Clear 逻辑

6. 关键技术注意事项：
   - TPair<FName, FName> 作为 TMap 的 Key 需要 GetTypeHash 支持
     UE4 的 TPair 默认支持 GetTypeHash（如果两个元素类型都支持）
     FName 支持 GetTypeHash，所以 TPair<FName, FName> 可直接用作 Key
   - 如果编译报 GetTypeHash 错误，改用自定义结构体：
     struct FEdgeKey { FName Source; FName Target; }
     并实现 operator== 和 friend GetTypeHash
   - 推导逻辑在 OnFlowEventReceived 中执行，时机正确：
     先更新 NodeLastActiveTime（P6.1），再推导 Edge（本任务）
   - SourceElapsed >= 0 的检查防止时间戳异常（理论上不会 <0）
   - 需要 #include "FlowPipelineRegistry.h"（如尚未包含）
