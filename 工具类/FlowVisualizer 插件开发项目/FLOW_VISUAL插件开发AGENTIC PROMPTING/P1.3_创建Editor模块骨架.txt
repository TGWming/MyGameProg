【具体要求】

1. 需要创建 3 个文件：

   文件 A：Plugins/FlowVisualizer/Source/FlowVisualizerEditor/FlowVisualizerEditor.Build.cs
   文件 B：Plugins/FlowVisualizer/Source/FlowVisualizerEditor/Public/FlowVisualizerEditorModule.h
   文件 C：Plugins/FlowVisualizer/Source/FlowVisualizerEditor/Private/FlowVisualizerEditorModule.cpp

2. Build.cs 应满足：
   - ModuleRules 类名：FlowVisualizerEditor
   - PCHUsage：UseExplicitOrSharedPCHs
   - PublicDependencyModuleNames：
     "Core", "CoreUObject", "Engine", "FlowVisualizerRuntime"
   - PrivateDependencyModuleNames：
     "Slate", "SlateCore", "UnrealEd", "EditorStyle", "LevelEditor", "InputCore"
   - 注意：依赖 FlowVisualizerRuntime 是因为 Editor 模块需要读取
     Runtime 模块提供的数据类型和单例

3. Module .h 应满足：
   - #pragma once
   - 包含 "Modules/ModuleManager.h"
   - 声明类继承 IModuleInterface
   - 声明 StartupModule() 和 ShutdownModule() 的 override

4. Module .cpp 应满足：
   - 实现 StartupModule()：打印日志 "FlowVisualizerEditor module loaded"
     使用 UE_LOG(LogTemp, Log, ...)
   - 实现 ShutdownModule()：打印日志 "FlowVisualizerEditor module unloaded"
   - 文件末尾使用 IMPLEMENT_MODULE 宏注册模块
     IMPLEMENT_MODULE(FFlowVisualizerEditorModule, FlowVisualizerEditor)

【此任务不做的事情】
- 不在 StartupModule() 中注册菜单项（那是 P3.1）
- 不创建 SFlowVisualizerPanel 或 SWindow 相关代码（那是 P3.2、P3.3）
- 不创建 FlowVisualizerStyle 样式文件（那是 P5.3）
- 不创建 Resources 目录或图标（那是 P1.4）

【验证标准】
- 文件编码 UTF-8
- 每个 .h 文件包含 #pragma once
- Build.cs 中 ModuleRules 类名与 .uplugin 中声明的模块名一致
- Build.cs 的 PublicDependencyModuleNames 包含 "FlowVisualizerRuntime"
- Build.cs 的 PrivateDependencyModuleNames 包含 "Slate", "SlateCore",
  "UnrealEd", "EditorStyle", "LevelEditor", "InputCore"
- 编译通过
- 编辑器启动后 Output Log 中出现 "FlowVisualizerEditor module loaded"
- 未创建本任务范围外的任何文件
