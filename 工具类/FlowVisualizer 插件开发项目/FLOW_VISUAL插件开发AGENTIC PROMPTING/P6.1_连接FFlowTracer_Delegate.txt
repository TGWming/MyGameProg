【任务目标】
在 SFlowVisualizerPanel 中监听 FFlowTracer 的 OnFlowEvent Delegate，
接收运行时事件并存储到节点状态映射表中。
实现采样节流逻辑（100ms 窗口内同节点不重复触发高亮重置）。
这是事件驱动可视化的数据接收层，后续 P6.2/P6.3 负责渲染。

【具体要求】

1. 需要修改的文件：

   文件 A：Plugins/FlowVisualizer/Source/FlowVisualizerEditor/Public/SFlowVisualizerPanel.h
   文件 B：Plugins/FlowVisualizer/Source/FlowVisualizerEditor/Private/SFlowVisualizerPanel.cpp

2. SFlowVisualizerPanel.h 中应新增以下成员声明：

   Delegate 句柄：
   - FDelegateHandle FlowEventHandle
     （监听 FFlowTracer::OnFlowEvent 的句柄）

   节点状态映射表：
   - TMap<FName, double> NodeLastActiveTime
     （Key: NodeID, Value: 最后一次激活的时间戳）
   - TMap<FName, FString> NodeLastData
     （Key: NodeID, Value: 最后一次事件携带的 DataPayload）

   回调函数：
   - void OnFlowEventReceived(const FFlowEvent& Event)
     （接收事件的回调）

3. Construct() 末尾应注册 Delegate：
   - FlowEventHandle = FFlowTracer::Get().OnFlowEvent.AddRaw(
       this, &SFlowVisualizerPanel::OnFlowEventReceived)
   - 需要 #include "FlowTracer.h"
   - 需要 #include "FlowPipelineTypes.h"（FFlowEvent 类型）

4. 析构/销毁时应取消 Delegate：
   - FFlowTracer::Get().OnFlowEvent.Remove(FlowEventHandle)
   - 注意：SCompoundWidget 没有虚析构函数直接可 override，
     可在自定义析构函数中处理，或使用
     FSlateApplication::Get().OnPreShutDown 等方式
   - 推荐方式：声明 ~SFlowVisualizerPanel() 析构函数，
     在其中移除 Delegate

5. OnFlowEventReceived() 实现逻辑：

   步骤 1 — 暂停检查：
   - 如果 bIsPaused == true，直接 return

   步骤 2 — 采样节流检查：
   - 查找 NodeLastActiveTime 中是否已有 Event.NodeID 的记录
   - 如果已有记录：
     double LastTime = NodeLastActiveTime[Event.NodeID]
     double Elapsed = Event.Timestamp - LastTime
     如果 Elapsed < FlowVisStyle::SignalThrottleMs / 1000.0
       （即 100ms 内重复信号）：
       仅更新数据：NodeLastData[Event.NodeID] = Event.DataPayload
       不更新时间戳（不重置高亮动画）
       return

   步骤 3 — 正常更新：
   - NodeLastActiveTime.Add(Event.NodeID, Event.Timestamp)
     或 NodeLastActiveTime[Event.NodeID] = Event.Timestamp
   - NodeLastData.Add(Event.NodeID, Event.DataPayload)
     或 NodeLastData[Event.NodeID] = Event.DataPayload

6. 关键技术注意事项：
   - OnFlowEvent Delegate 可能从非 GameThread 广播
     （取决于 Signal() 的调用线程），但本项目中 Signal() 通常
     在 GameThread 调用，暂不需要线程安全处理
   - AddRaw 绑定的是裸指针，析构时必须 Remove，否则悬挂指针崩溃
   - FlowVisStyle::SignalThrottleMs 已在 P5.3 中定义为 100.0f
   - NodeLastActiveTime 和 NodeLastData 在后续 P6.2/P6.3 中被
     OnPaint 读取，用于渲染高亮和数据标签

【此任务不做的事情】
