【任务目标】
在 FlowVisualizerRuntime 模块的 Public 目录下创建 FlowTracerMacros.h，
提供 3 个便捷宏供游戏代码埋点使用。
关键特性：在 Shipping 构建中宏展开为空操作（零开销）。

【具体要求】

1. 目标文件：
   Plugins/FlowVisualizer/Source/FlowVisualizerRuntime/Public/FlowTracerMacros.h

2. 文件结构应满足：
   - #pragma once
   - #include "FlowTracer.h"
   - 使用 #if !UE_BUILD_SHIPPING 条件编译

3. 非 Shipping 构建时，定义 3 个宏：

   FLOW_SIGNAL(NodeID, Data)
   - 调用 FFlowTracer::Get().Signal(FName(TEXT(NodeID)), Data)
   - 用途：发送带数据的事件
   - 示例：FLOW_SIGNAL("Movement", SomeDataString)

   FLOW_SIGNAL_SIMPLE(NodeID)
   - 调用 FFlowTracer::Get().Signal(FName(TEXT(NodeID)))
   - 用途：发送不带数据的简单事件
   - 示例：FLOW_SIGNAL_SIMPLE("PlayerInput")

   FLOW_SIGNAL_FMT(NodeID, Fmt, ...)
   - 调用 FFlowTracer::Get().Signal(FName(TEXT(NodeID)),
         FString::Printf(TEXT(Fmt), ##__VA_ARGS__))
   - 用途：发送格式化数据的事件
   - 示例：FLOW_SIGNAL_FMT("Combat", "Type=%s Dmg=%d", *TypeStr, Damage)

4. Shipping 构建时（#else 分支）：
   - 3 个宏全部定义为空操作（无展开内容）
   - 确保零运行时开销

5. 关键技术注意事项：
   - 可变参数宏使用 ##__VA_ARGS__ 语法（兼容 MSVC 和 GCC/Clang）
   - NodeID 参数为字面字符串，宏内部用 TEXT() 包裹转换
   - Data 参数为 FString 类型，调用侧负责构造
