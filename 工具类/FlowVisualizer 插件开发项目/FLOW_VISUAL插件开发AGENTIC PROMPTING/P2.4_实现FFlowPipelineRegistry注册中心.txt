【任务目标】
在 FlowVisualizerRuntime 模块中实现 FFlowPipelineRegistry 单例类，
作为管线定义的全局注册中心。
游戏代码和编辑器模块均通过此注册中心注册/查询管线拓扑定义。
注意：Registry 放在 Runtime 模块中，以便游戏代码也能注册管线。

【具体要求】

1. 需要创建 2 个文件：

   文件 A：Plugins/FlowVisualizer/Source/FlowVisualizerRuntime/Public/FlowPipelineRegistry.h
   文件 B：Plugins/FlowVisualizer/Source/FlowVisualizerRuntime/Private/FlowPipelineRegistry.cpp

2. FFlowPipelineRegistry 类的公开接口应包含：

   static FFlowPipelineRegistry& Get();
   - Meyer's Singleton 模式（与 FFlowTracer 一致）

   void RegisterPipeline(const FFlowPipeline& Pipeline);
   - 以 Pipeline.PipelineID 为 Key 存入内部 TMap
   - 如果相同 ID 已存在则覆盖
   - 存入后广播 OnRegistryChanged Delegate

   void UnregisterPipeline(FName PipelineID);
   - 从 TMap 中移除指定 ID 的管线
   - 移除后广播 OnRegistryChanged Delegate

   const FFlowPipeline* FindPipeline(FName PipelineID) const;
   - 从 TMap 中查找并返回指针
   - 未找到返回 nullptr

   TArray<FName> GetAllPipelineIDs() const;
   - 返回所有已注册管线的 PipelineID 列表

   Delegate 声明：
   DECLARE_MULTICAST_DELEGATE(FOnRegistryChanged);
   FOnRegistryChanged OnRegistryChanged;

3. FFlowPipelineRegistry 类的私有成员应包含：

   TMap<FName, FFlowPipeline> RegisteredPipelines;
   - 以 PipelineID 为 Key 的管线存储

4. 关键技术注意事项：
   - 类需要添加 FLOWVISUALIZERRUNTIME_API 导出宏
   - 需要 #include "FlowPipelineTypes.h" 获取 FFlowPipeline 类��
   - OnRegistryChanged 供编辑器面板监听，当管线注册/注销时
     刷新下拉菜单列表（后续 P4.2 会连接此 Delegate）
   - Registry 不需要线程锁（管线注册通常在 GameThread 进行）
