【任务目标】
完善 SFlowVisualizerPanel 中 Pause 和 Clear 按钮的完整交互逻辑。
Pause 按钮实现文字切换（Pause ↔ Resume）和暂停时底栏提示。
Clear 按钮实现全部状态映射表的清空和视觉重置。

【具体要求】

1. 需要修改的文件：

   文件 A：Plugins/FlowVisualizer/Source/FlowVisualizerEditor/Public/SFlowVisualizerPanel.h
   文件 B：Plugins/FlowVisualizer/Source/FlowVisualizerEditor/Private/SFlowVisualizerPanel.cpp

2. Pause 按钮完善：

   当前状态（P4.2 已实现）：
   - 按钮文字固定为 "Pause"
   - 点击切换 bIsPaused 标志

   需要改为：
   - 按钮文字动态显示：
     bIsPaused == false 时显示 "Pause"
     bIsPaused == true 时显示 "Resume"
   - 实现方式：SButton 的 Text 属性绑定为动态获取：
     使用 SButton Content 中的 STextBlock，其 Text 属性绑定
     TAttribute<FText> 或 Lambda，根据 bIsPaused 返回对应文字

   暂停时的行为确认（P6.1 已实现）：
   - bIsPaused = true 时 OnFlowEventReceived 不处理新事件
   - 当前显示冻结（已有的高亮和数据标签保持当前状态）
   - Tick 仍然运行（FPS 继续更新），但不接收新事件

   暂停时底栏提示：
   - 在 OnPaint 底栏绘制中，如果 bIsPaused == true：
     在状态文字末尾追加 " | PAUSED"
     "PAUSED" 文字使用 FlowVisStyle::Warning 颜色（#FF6644 橙红）
   - 实现方式：可以在底栏文字拼接时条件追加，
     或单独绘制一个 "PAUSED" 文字覆盖在底栏右侧

3. Clear 按钮完善：

   当前状态（P4.2 已实现）：
   - 点击调用 FFlowTracer::Get().ClearBuffer()

   需要追加清空所有视觉状态：
   - NodeLastActiveTime.Empty()
   - NodeLastData.Empty()
   - EdgeLastActiveTime.Empty()
   - ActiveSignalCount = 0
   - LastSignalElapsed = 0.0

   清空后的视觉效果：
   - 所有节点回到暗灰色 FlowVisStyle::NodeInactive
   - 所有数据标签消失
   - 所有连线回到默认灰色 FlowVisStyle::EdgeDefault
   - 所有流动光点消失
   - 触发一次 Invalidate 确保立即重绘

   实现建议：
   - 将 Clear 逻辑提取为独立函数 ClearVisualization()
   - Clear 按钮回调中先调用 FFlowTracer::Get().ClearBuffer()
     再调用 ClearVisualization()
   - ClearVisualization() 末尾调用
     Invalidate(EInvalidateWidget::Paint)（或对应 UE4.27 API）

4. 关键技术注意事项：
   - SButton 的动态文字可通过以下方式实现：
     方式 A：SButton 内嵌 STextBlock，STextBlock 的 Text 使用
       TAttribute 绑定到 Lambda 或成员函数
     方式 B：使用 SButton 的 .Text() 属性（如果支持 TAttribute）
   - "PAUSED" 提示需要使用不同颜色，如果底栏是单次 MakeText 调用，
     可能需要拆分为两次 MakeText：一次常规状态文字，一次 PAUSED 标记
   - 或者直接将 "PAUSED" 拼入状态文字字符串，但颜色统一为 Warning
