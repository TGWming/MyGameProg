【项目背景】
FlowVisualizer 是一个 UE 4.27 编辑器插件，用于在编辑器独立弹出窗口中
以终端风格可视化观察运行时数据流。
插件采用双模块结构：FlowVisualizerRuntime + FlowVisualizerEditor。

【当前进度】
Phase 1 P1.1（共 32 个原子化任务中的第 1 个）

前置完成：
- 无（P1.1 是整个任务图的起点）

2. 禁止修改项目 .uproject、项目 Build.cs 或项目任何已有源文件
3. 禁止引入第三方库；仅使用 UE 4.27 原生 API
4. 所有运行时代码必须包裹在 Shipping 空操作保护中
5. 不产生任何 UI 到游戏视口；仅限编辑器独立窗口
6. 每个 Task 结束后必须能编译通过且不影响现有项目

【任务目标】
在插件根目录创建 .uplugin 描述文件，声明双模块结构，
使 UE 4.27 能正确识别并加载此插件。

【具体要求】

1. 目标文件：Plugins/FlowVisualizer/FlowVisualizer.uplugin

2. 该文件应包含以下内容：
   - FileVersion: 3（UE 4.27 标准）
   - Version: 1
   - VersionName: "0.1.0"
   - FriendlyName: "FlowVisualizer"
   - Description: "Terminal-style runtime data-flow visualizer for editor observation."
   - Category: "Developer Tools"
   - CreatedBy: "TGWming"
   - EnabledByDefault: true
   - CanContainContent: false（纯代码插件，不含资产）
   - IsBetaVersion: true

3. Modules 数组应声明恰好 2 个模块：
   - FlowVisualizerRuntime
     Type: "Runtime"
     LoadingPhase: "Default"
   - FlowVisualizerEditor
     Type: "Editor"
     LoadingPhase: "Default"

4. 关键说明：
   - Runtime 模块承载运行时数据采集（FFlowTracer / 宏 / 管线结构）
   - Editor 模块承载编辑器窗口、渲染、菜单注册，不打包进 Shipping
   - LoadingPhase 均为 "Default"（无需 PreDefault 或 PostDefault）

【此任务不做的事情】
- 不创建任何 .h / .cpp / .Build.cs 文件（那是 P1.2、P1.3）
- 不创建 Resources 目录或图标（那是 P1.4）
- 不创建任何目录结构骨架以外的内容

【验证标准】
- 文件位于 Plugins/FlowVisualizer/FlowVisualizer.uplugin
- JSON 格式合法（可用任意 JSON 校验器确认）
- FileVersion 为 3
- Modules 数组包含恰好 2 个模块
- FlowVisualizerRuntime 的 Type 为 "Runtime"
- FlowVisualizerEditor 的 Type 为 "Editor"
- 未创建本任务范围外的任何文件
