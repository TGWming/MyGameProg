【任务目标】
在 FlowVisualizerRuntime 模块的 Public 目录下创建 FlowPipelineTypes.h，
定义插件所有核心数据结构体。这些结构体是整个插件数据流的基础类型。

【具体要求】

1. 目标文件：
   Plugins/FlowVisualizer/Source/FlowVisualizerRuntime/Public/FlowPipelineTypes.h

2. 需要定义 4 个结构体：

   结构体 A — FFlowEvent（运行时事件）：
   - double Timestamp           // 事件时间戳，使用 FPlatformTime::Seconds()
   - uint64 FrameNumber         // 事件发生帧号，使用 GFrameNumber
   - FName NodeID               // 触发该事件的节点 ID，如 "Movement"
   - FString DataPayload        // 自由格式数据字符串

   结构体 B — FFlowNode（管线节点）：
   - FName NodeID               // 节点唯一标识
   - FText DisplayName          // 节点显示名称
   - FVector2D RelativePosition // 节点在画布上的相对位置，范围 (0~1)

   结构体 C — FFlowEdge（管线连线）：
   - FName SourceNodeID         // 连线起点节点 ID
   - FName TargetNodeID         // 连线终点节点 ID

   结构体 D — FFlowPipeline（管线定义）：
   - FName PipelineID           // 管线唯一标识
   - FText DisplayName          // 管线显示名称
   - TArray<FFlowNode> Nodes    // 管线包含的所有节点
   - TArray<FFlowEdge> Edges    // 管线包含的所有连线

3. 所有结构体应满足：
   - 添加 FLOWVISUALIZERRUNTIME_API 导出宏（确保外部模块可使用）
   - 提供默认构造函数
   - #pragma once 在文件头部
   - 包含必要的 UE4 头文件（CoreMinimal.h 等）

4. 关于 API 导出宏：
   - 如果 FlowVisualizerRuntime.Build.cs 中尚未自动生成
     FLOWVISUALIZERRUNTIME_API 宏，需要确认 Build.cs 中是否需要
     添加对应的设置（UE4.27 模块构建系统通常自动生成）
   - 如果自动生成不可用，在头文件中用条件编译处理

【此任务不做的事情】
