【任务目标】
填充 SFlowVisualizerPanel 的 Tick() 实现，在每帧检查是否有
活跃的动画（节点高亮渐灭或 Edge 动画），如果有则调用
Invalidate 触发下一帧 OnPaint 重绘。
无活跃动画时不调用 Invalidate，节省 CPU。
同时更新底栏的 ActiveSignals 统计数据。

【具体要求】

1. 需要修改的文件：

   文件 A：Plugins/FlowVisualizer/Source/FlowVisualizerEditor/Public/SFlowVisualizerPanel.h
   文件 B：Plugins/FlowVisualizer/Source/FlowVisualizerEditor/Private/SFlowVisualizerPanel.cpp

2. SFlowVisualizerPanel.h 中应新增：
   - int32 ActiveSignalCount = 0
     （当前活跃节点数量，底栏显示用）
   - double LastSignalElapsed = 0.0
     （最近一次信号距今时间，底栏显示用，单位毫秒）

3. Tick() 实现逻辑（替换 P3.2 中的空实现）：

   步骤 1 — 调用父类 Tick：
   - SCompoundWidget::Tick(AllottedGeometry, InCurrentTime, InDeltaTime)

   步骤 2 — FPS 缓存更新（保留 P4.3 已实现的逻辑）：
   - 如果 InDeltaTime > 0：
     CachedFPS = FMath::Lerp(CachedFPS, 1.0f / InDeltaTime, 0.1f)

   步骤 3 — 检查活跃节点动画：
   - double CurrentTime = FPlatformTime::Seconds()
   - bool bHasActiveAnimation = false
   - int32 TempActiveCount = 0
   - double TempMinElapsed = TNumericLimits<double>::Max()

   - 遍历 NodeLastActiveTime：
     for (const auto& Pair : NodeLastActiveTime)
       double Elapsed = CurrentTime - Pair.Value
       如果 Elapsed < FlowVisStyle::HighlightFadeDuration：
         bHasActiveAnimation = true
         TempActiveCount++
         TempMinElapsed = FMath::Min(TempMinElapsed, Elapsed)

   步骤 4 — 检查活跃 Edge 动画（为 P7.x 预留）：
   - 留注释占位：
     // TODO P7.1/P7.2: 检查 EdgeLastActiveTime 中的活跃 Edge
     // 如果有活跃 Edge，bHasActiveAnimation = true

   步骤 5 — 更新统计数据：
   - ActiveSignalCount = TempActiveCount
   - 如果 TempActiveCount > 0：
     LastSignalElapsed = TempMinElapsed * 1000.0（转毫秒）
   否则：
     LastSignalElapsed = 0.0

   步骤 6 — 智能 Invalidate：
   - 如果 bHasActiveAnimation：
     Invalidate(EInvalidateWidget::Paint)
     // 注意：UE4.27 中可能是 EInvalidateWidget::Paint
     // 或 EInvalidateWidgetReason::Paint，需确认 API
     // ��果两者都不可用，可使用 Invalidate(EInvalidateWidget::LayoutAndVolatility)
   - 如果 !bHasActiveAnimation：
     不调用 Invalidate（节省 CPU，面板静止）

4. 底栏状态文字扩展（可选，也可留到 P9.2）：
   - 如果在本任务中方便，可在 OnPaint 底栏文字中追加 ActiveSignalCount
   - 格式调整为："Frame: {N} | FPS: {N} | Active: {N} | Pipeline: {name}"
   - 如果不方便，留注释说明 P9.2 会扩展，保持现有格式

5. 关键技术注意事项：
   - UE4.27 中 SWidget::Invalidate 的参数可能是
     EInvalidateWidget 或 EInvalidateWidgetReason 枚举，
     具体取决于引擎小版本。如果编译不过，尝试：
     a. Invalidate(EInvalidateWidget::Paint)
     b. Invalidate(EInvalidateWidgetReason::Paint)
     c. 或直接不传参数 Invalidate()（部分版本支持）
   - Tick 每帧调用，遍历 NodeLastActiveTime 的开销很低
     （通常只有几个到几十个节点）
   - 无活跃动画时不 Invalidate 是关键优化，
     避免编辑器空闲时无意义的重绘开销
