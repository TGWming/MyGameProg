【任务目标】
修改 SFlowVisualizerPanel OnPaint() 中的节点绘制逻辑，
根据 NodeLastActiveTime 计算每个节点距离上次激活的时间差，
实现从亮青色（#33CCFF）到暗灰色（#666666）的 1.5 秒渐灭过渡。
信号触发时节点文字瞬间变亮，随后平滑衰减至不活跃状态。

【具体要求】

1. 需要修改的文件：

   Plugins/FlowVisualizer/Source/FlowVisualizerEditor/Private/SFlowVisualizerPanel.cpp

2. 修改 OnPaint() 中节点绘制部分的颜色计算逻辑：

   对每个 FFlowNode，在绘制前计算动态颜色：

   步骤 1 — 获取当前时间：
   - double CurrentTime = FPlatformTime::Seconds()

   步骤 2 — 查找节点最后激活时间：
   - const double* LastActivePtr = NodeLastActiveTime.Find(Node.NodeID)

   步骤 3 — 计算颜色：
   - 如果 LastActivePtr 不为 nullptr：
     double Elapsed = CurrentTime - *LastActivePtr
     如果 Elapsed < FlowVisStyle::HighlightFadeDuration（1.5 秒）：
       float Alpha = 1.0f - (float)(Elapsed / FlowVisStyle::HighlightFadeDuration)
       TextColor = FMath::Lerp(FlowVisStyle::NodeInactive, FlowVisStyle::NodeActive, Alpha)
     否则：
       TextColor = FlowVisStyle::NodeInactive
   - 如果 LastActivePtr 为 nullptr（节点从未被激活）：
     TextColor = FlowVisStyle::NodeInactive

   步骤 4 — 使用计算出的 TextColor 绘制节点文字：
   - 替换原来固定的 FlowVisStyle::NodeInactive
   - 其余绘制参数不变（字体、位置、格式）

3. 关键技术注意事项：
   - FMath::Lerp 对 FLinearColor 有直接重载，可以直接使用
   - Alpha 从 1.0（刚激活，全亮）平滑过渡到 0.0（渐灭完成，全暗）
   - Lerp 参数顺序：Lerp(A, B, Alpha) 当 Alpha=1 时返回 B（ActiveColor）
     当 Alpha=0 时返回 A（InactiveColor）
   - CurrentTime 应在 OnPaint 入口处计算一次，所有节点共用
     （避免同帧内不同节点时间不一致）
   - HighlightFadeDuration 已在 P5.3 中定义为 1.5f
   - 注意：本任务只修改颜色计算，不修改 Tick 刷新逻辑（P6.4 负责）
     在 P6.4 完成前，动画效果需要手动触发重绘才能观察到
