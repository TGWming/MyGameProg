   # FlowVisualizer

   ## 概述
   - 一句话描述：UE4.27 编辑器插件，以终端风格可视化观察运行时数据流
   - 适用场景：开发调试期间观��游戏系统间的数据流转和时序关系
   - 特点：零 Shipping 开销、独立编辑器窗口、实时动画

   ## 快速开始
   - 步骤 1：将 FlowVisualizer 文件夹放入项目的 Plugins/ 目录
   - 步骤 2：在游戏模块的 Build.cs 中添加 "FlowVisualizerRuntime" 依赖
   - 步骤 3：在游戏代码中 #include "FlowTracerMacros.h"
   - 步骤 4：在需要监控的代码位置添加 FLOW_SIGNAL 调用
   - 步骤 5：定义并注册管线拓扑（FFlowPipeline + FFlowPipelineRegistry）
   - 步骤 6：编辑器菜单 Window → Flow Visualizer 打开窗口

   ## 宏 API
   - FLOW_SIGNAL(NodeID, Data)
     发送带数据的事件
     示例：FLOW_SIGNAL("Movement", SomeDataString)
   - FLOW_SIGNAL_SIMPLE(NodeID)
     发送不带数据的简单事件
     示例：FLOW_SIGNAL_SIMPLE("PlayerInput")
   - FLOW_SIGNAL_FMT(NodeID, Fmt, ...)
     发送格式化数据的事件
     示例：FLOW_SIGNAL_FMT("Combat", "Dmg=%d Type=%s", Damage, *TypeStr)
   - 说明：所有宏在 Shipping 构建中展开为空操作，零运行时开销

   ## 管线注册
   - FFlowPipeline 结构说明（PipelineID, Nodes, Edges）
   - FFlowPipelineRegistry::Get().RegisterPipeline() 用法
   - 代码示例：注册一个简单的 3 节点管线

   ## 窗口操作
   - 顶栏：Pipeline 下拉菜单切换管线视图
   - Pause 按钮：暂停/恢复事件接收
   - Clear 按钮：清空所有状态和缓冲区
   - 底栏：Frame / FPS / Active / Last / Buffer 统计信息
   - 窗口位置和大小自动记忆

   ## 视觉规格
   - 节点：白色方括号 "[NodeName]" 终端风格
   - 节点高亮：信号触发时亮青色 #33CCFF → 1.5 秒渐灭 → 暗灰色 #666666
   - 数据标签：绿色 #4AFF82 Mono 字体，高亮期间显示
   - 连线：灰色 #888888 + 箭头，激活时亮青色 #33CCFF 渐灭
   - 流动光点：白色 6x6 方块沿激活连线移动
   - 背景：深蓝黑 #0D0D14

   ## 架构
   - FlowVisualizerRuntime 模块：
     FlowPipelineTypes.h — 核心数据结构体
     FlowTracer.h — 事件采集单例
     FlowTracerMacros.h — ���捷宏
     FlowPipelineRegistry.h — 管线注册中心
   - FlowVisualizerEditor 模块：
     FlowVisualizerEditorModule — 菜单注册 + 窗口管理
     SFlowVisualizerPanel — 主画布 Widget
     FlowVisualizerStyle — 颜色/字体/动画常量

   ## 内���管线
   - Demo：3 节点 2 连线，用于测试
   - Soul 项目管线（4 条）：
     CharacterMovement, LockOnSystem, CameraPipeline, CombatSystem

   ## 技术说明
   - UE 版本：4.27
   - 线程安全：FFlowTracer 使用 FCriticalSection 保护缓冲区
   - 性能：无活跃动画时 Tick 不触发 Invalidate，零 CPU 空闲开销
   - Shipping：所有 FLOW_SIGNAL 宏在 Shipping 构建中展开为空
