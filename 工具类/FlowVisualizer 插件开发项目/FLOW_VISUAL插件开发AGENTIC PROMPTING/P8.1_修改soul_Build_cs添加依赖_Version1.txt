【项目背景】
FlowVisualizer 是一个 UE 4.27 编辑器插件，用于在编辑器独立弹出窗口中
以终端风格可视化观察运行时数据流。
插件采用双模块结构：FlowVisualizerRuntime + FlowVisualizerEditor。

本任务开始 Phase 8（Soul 项目接入），将 FlowVisualizer 插件
与现有的 Soul 类魂游戏项目对接。

【任务目标】
修改 Soul 项目的 Build.cs 文件，在 PublicDependencyModuleNames 中
添加 "FlowVisualizerRuntime"，使游戏代码可以
#include "FlowTracerMacros.h" 并使用 FLOW_SIGNAL 宏。

【具体要求】

1. 需要修改的文件：

   soul.Build.cs
   （位于 Source/soul/ 目录下，具体路径以项目实际结构为准）

2. 修改内容：
   - 在 PublicDependencyModuleNames.AddRange 调用中追加
     "FlowVisualizerRuntime"
   - 修改后应类似：
     PublicDependencyModuleNames.AddRange(new string[] {
         "Core", "CoreUObject", "Engine", "InputCore", "UMG",
         "FlowVisualizerRuntime"
     });
   - 注意：保留原有的所有依赖项，仅追加一项

3. 关键说明：
   - 仅添加 Runtime 模块依赖，不添加 Editor 模块依赖
     （游戏代码只需要 Runtime 提供的宏和类型）
   - FlowVisualizerRuntime 模块通过 FLOW_SIGNAL 宏提供
     Shipping 空操作保护，添加依赖不会影响最终发布包大小
   - 如果 Build.cs 中已有条件编译分支（如 #if WITH_EDITOR），
     FlowVisualizerRuntime 依赖应放在无条件的
     PublicDependencyModuleNames 中（因为宏在 Runtime 也需要编译）
