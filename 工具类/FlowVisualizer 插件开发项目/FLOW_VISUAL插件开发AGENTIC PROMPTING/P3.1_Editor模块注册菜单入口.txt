插件采用双模块结构：FlowVisualizerRuntime + FlowVisualizerEditor。

【任务目标】
修改 FlowVisualizerEditorModule 的 StartupModule() 和 ShutdownModule()，
在 UE4 编辑器的 Window 菜单下注册 "Flow Visualizer" 菜单入口。
使用 UE4.27 的 FExtender + FLevelEditorModule 方式（非 UToolMenus）。

【具体要求】

1. 需要修改的文件：

   文件 A：Plugins/FlowVisualizer/Source/FlowVisualizerEditor/Public/FlowVisualizerEditorModule.h
   文件 B：Plugins/FlowVisualizer/Source/FlowVisualizerEditor/Private/FlowVisualizerEditorModule.cpp

2. FlowVisualizerEditorModule.h 中应新增以下成员声明：
   - TSharedPtr<FUICommandList> PluginCommands
   - TSharedPtr<FExtender> MenuExtender
   - void OpenVisualizerWindow()
     （本任务中仅打印日志占位，实际窗口弹出在 P3.3 实现）

3. StartupModule() 中应实现：
   - 创建 FUICommandList 并映射一个 OpenWindow 动作
   - 创建 FExtender
   - 使用 AddMenuExtension 注册到 "WindowLayout" 位置
     Hook 类型：EExtensionHook::After
   - 获取 FLevelEditorModule（通过 FModuleManager::LoadModuleChecked）
   - 调用 FLevelEditorModule 的 GetMenuExtensibilityManager()->AddExtender(...)
   - 菜单项属性：
     标题："Flow Visualizer"
     提示："Open runtime data flow visualization window"
     点击回调：调用 OpenVisualizerWindow()

4. ShutdownModule() 中应实现���
   - 如果 MenuExtender 有效，从 LevelEditor 的
     MenuExtensibilityManager 中移除
   - 重置 PluginCommands 和 MenuExtender

5. OpenVisualizerWindow() 本任务中的临时实现：
   - 仅打印日志：UE_LOG(LogTemp, Log, TEXT("FlowVisualizer: OpenVisualizerWindow called"))
   - 后续 P3.3 会将此替换为实际的 SWindow 弹出逻辑

6. 关键技术注意事项：
   - UE4.27 使用 FExtender 而非 UToolMenus 注册编辑器菜单
   - AddMenuExtension 的第一个参数 "WindowLayout" 是 Window 菜单
     中的 hook 位置名称
   - FLevelEditorModule 需要 #include "LevelEditor.h"
   - FUICommandList 需要 #include "Framework/Commands/UICommandList.h"
   - FExtender 需要 #include "Framework/MultiBox/MultiBoxExtender.h"

【此任务不做的事情】
- 不创建 SFlowVisualizerPanel Widget（那是 P3.2）
- 不实现 SWindow 弹出逻辑（那是 P3.3）
- 不创建任何新文件（仅修改已有的 Module .h/.cpp）
- 不注册任何 Demo 管线（那是 P4.1）
