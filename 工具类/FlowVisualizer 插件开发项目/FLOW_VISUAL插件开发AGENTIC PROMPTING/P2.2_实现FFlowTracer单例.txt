【任务目标】
在 FlowVisualizerRuntime 模块中实现 FFlowTracer 单例类，
作为整个插件运行时事件采集的核心枢纽。
游戏代码通过 FFlowTracer::Get().Signal() 发送事件，
编辑器面板通过 Delegate 监听和读取事件。

【具体要求】

1. 需要创建 2 个文件：

   文件 A：Plugins/FlowVisualizer/Source/FlowVisualizerRuntime/Public/FlowTracer.h
   文件 B：Plugins/FlowVisualizer/Source/FlowVisualizerRuntime/Private/FlowTracer.cpp

2. FFlowTracer 类的公开接口应包含：

   static FFlowTracer& Get();
   - Meyer's Singleton 模式（函数内静态变量）

   void Signal(FName NodeID, const FString& Data = TEXT(""));
   - 构造 FFlowEvent：
     Timestamp = FPlatformTime::Seconds()
     FrameNumber = GFrameNumber
     NodeID = 传入参数
     DataPayload = 传入参数
   - 加锁写入 EventBuffer（环形缓冲逻辑）
   - 在锁外广播 OnFlowEvent Delegate（重要：避免死锁）

   TArray<FFlowEvent> GetRecentEvents(int32 Count) const;
   - 从 EventBuffer 中取最近 Count 条事件
   - 加锁读取

   void ClearBuffer();
   - 清空 EventBuffer，重置 BufferHead
   - 加锁

   Delegate 声明：
   DECLARE_MULTICAST_DELEGATE_OneParam(FOnFlowEvent, const FFlowEvent&);
   FOnFlowEvent OnFlowEvent;

3. FFlowTracer 类的私有成员应包含：

   TArray<FFlowEvent> EventBuffer;
   - 用 TArray 模拟环形缓冲区
   - 初始化时 SetNum(BufferCapacity)

   int32 BufferHead = 0;
   - 当前写入位置

   int32 BufferCapacity = 1024;
   - 环形缓冲区容量

   FCriticalSection BufferLock;
   - 防御式线程安全锁

4. 关键技术注意事项：
   - UE4.27 中 TCircularBuffer 签名可能与预期不同，
     使用 TArray + Head 指针模拟环形缓冲更安全
   - Signal() 中 OnFlowEvent.Broadcast() 必须在锁外调用，
     避免 Delegate 回调中再次获取锁导致死锁
   - 环形写入逻辑：写入 EventBuffer[BufferHead]，
     然后 BufferHead = (BufferHead + 1) % BufferCapacity
   - GetRecentEvents() 需要正确处理环形缓冲的读取顺序，
     返回按时间从旧到新排列的事件
   - 类需要添加 FLOWVISUALIZERRUNTIME_API 导出宏
