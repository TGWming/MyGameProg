# Steam / PS5 发行平台性能标准与低配测试基准

> 适用于：UE5 单机类魂/恐怖生存游戏 | 独立开发 | PS5 + Steam 双平台
> 数据来源：Steam Hardware Survey 2025-2026 / Sony PlayStation 认证标准 / UE5 性能实践
> 生成日期：2026-03-17

## 一、当前玩家硬件分布概况

### 1.1 Steam 硬件调查 (2025-2026)

#### GPU 分布

| GPU 级别 | 代表型号 | Steam 占比(约) | 你的关注度 |
|---|---|---|---|
| **当前主流** | RTX 3060 / RTX 4060 | 各 ~8-9% (合计 ~17%) | 中配目标 |
| **上一代主流** | RTX 2060 / GTX 1650 | 各 ~4-5% (合计 ~9%) | 推荐配置下限 |
| **老一代** | GTX 1060 / RX 580 | ~2-3% | ⚠️ 最低配置目标 |
| **更老/集显** | GTX 1050Ti及以下/Intel UHD | ~10-15% 合计 | ❌ 可以放弃 |
| **高端** | RTX 4070+ / RTX 3080+ | ~15-20% 合计 | 高配目标 |

> **关键数据：** GTX 1060 及以下（含同级AMD和集显）合计约占 Steam 用户的 **15-20%**。
> 如果你的最低配置定在 GTX 1060 级别，你将覆盖约 **80-85%** 的 Steam 用户。

#### CPU 分布

| CPU 级别 | 特征 | Steam 占比(约) |
|---|---|---|
| 6 核 | i5-9xxx ~ i5-12xxx / Ryzen 5 系列 | ~28-30% (最多) |
| 8 核 | i7-10xxx+ / Ryzen 7 系列 | ~25-28% (快速增长) |
| 4 核 | i5-7xxx 及更早 / Ryzen 3 | ~20% (持续下降) |
| 其他 | 2核/12核+ | ~15-20% |

#### 内存分布

| RAM | Steam 占比(约) |
|---|---|
| 16 GB | ~50%+ (绝对主流) |
| 32 GB | ~20%+ (快速增长) |
| 8 GB | ~15% (持续下降) |
| 8 GB 以下 | <5% |

#### 系统分辨率

| 分辨率 | 占比(约) |
|---|---|
| 1920×1080 (1080p) | ~58% (绝对主流) |
| 2560×1440 (1440p) | ~20% |
| 3840×2160 (4K) | ~5% |
| 其他(含超宽) | ~17% |

### 1.2 PS5 硬件规格（固定硬件）

| 组件 | PS5 | PS5 Pro |
|---|---|---|
| CPU | Zen 2, 8核 3.5GHz | Zen 2, 8核 3.85GHz |
| GPU | RDNA 2, 10.28 TFLOPS | RDNA 3/4, 16.7 TFLOPS |
| GPU 约等于 PC | RTX 2070 ~ RTX 2080 级 | RTX 3070 ~ RTX 3080 级 |
| RAM | 16 GB GDDR6 | 16 GB GDDR6 (更高带宽) |
| 存储 | 定制 SSD 825GB (5.5GB/s) | 定制 SSD 2TB |
| 输出 | 最高 4K@120Hz / 8K@60Hz | 4K@120Hz + PSSR 超分 |

> **关键结论：** PS5 基础版的 GPU 性能约等于 PC 的 RTX 2070。如果你的游戏在 PC 的 GTX 1060 上能以低画质运行，在 PS5 上跑 4K@30fps 或 1440p@60fps 完全没有问题。

## 二、你的游戏应该定什么配置标准

### 2.2 PS5 性能目标

| 模式 | 分辨率 | 帧率 | 画质 |
|---|---|---|---|
| **性能模式 (Performance)** | 1440p (动态) | **60 fps 稳定** | Medium-High |
| **画质模式 (Quality)** | 4K (原生或PSSR超分) | **30 fps 稳定** | Ultra + 光追可选 |

#### Sony PSN 认证硬性要求

| 要求 | 标准 | 说明 |
|---|---|---|
| **帧率稳定性** | 无持续掉帧、无严重卡顿 | 偶发单帧抖动可接受，连续掉帧到20fps以下会被拒 |
| **分辨率** | 必须支持4K输出显示 | 内部渲染可低于4K，但输出必须正确 |
| **双模式** | 强烈建议提供 Performance/Quality 切换 | 2025年后几乎是标配 |
| **不崩溃** | 全程无Crash/无死锁/无内存泄漏 | 最基本要求，QA必须覆盖 |
| **DualSense** | 必须支持基础震动 | 触觉反馈和自适应扳机为加分项(非必须) |
| **SSD加载** | 建议加载时间控制在合理范围 | PS5 SSD很快，但仍需优化Level Streaming |
| **PS5 Pro Enhanced (可选)** | 60fps + 更高分辨率/光追 | 获得"PS5 Pro Enhanced"标签需满足额外要求 |

## 三、UE5 类魂项目的性能优化策略

### 3.1 对你的项目特别重要的优化点

#### Lumen（全局光照）

| 策略 | 说明 |
|---|---|
| **恐怖游戏需要 Lumen** | 动态光照 + 实时GI 是恐怖氛围的核心（火把、手电筒、闪烁灯光） |
| **低配替代方案** | Low画质关闭 Lumen，改用传统 Screen Space GI + 烘焙光照 |
| **中高配** | Lumen Software Ray Tracing（不需要硬件光追） |
| **PS5** | Lumen 在 PS5 上表现良好，可以直接用 |

#### Nanite（虚拟几何体）

| 策略 | 说明 |
|---|---|
| **箱庭关卡受益大** | 中世纪/哥特建筑的大量重复石雕细节非常适合 Nanite |
| **注意** | Nanite 不支持骨骼网格体(Skeletal Mesh)，只用于静态场景 |
| **低配** | Nanite 自带 LOD，理论上对低配也有帮助，但需要 DX12 + 一定 GPU |

#### Virtual Shadow Maps (VSM)

| 策略 | 说明 |
|---|---|
| **配合 Lumen 使用** | 恐怖游戏大量使用点光源，VSM 处理多光源阴影更高效 |
| **低配替代** | 传统 Shadow Maps + 降低阴影分辨率 |

### 3.2 Scalability 分级建议

```
[Low] — GTX 1060 级别 / 1080p / 30fps 目标
├── Lumen: OFF → Screen Space GI + 部分烘焙
├── Nanite: ON (自带LOD优化)
├── Shadows: Medium, 2048 Resolution
├── Post-Process: Reduced (保留基本恐怖氛围效果)
├── View Distance: Medium
├── Foliage: Low
└── Texture: Medium (1K-2K)

[Medium] — RTX 2060 级别 / 1080p / 60fps 目标
├── Lumen: ON (Software)
├── Nanite: ON
├── Shadows: High, VSM
├── Post-Process: High
├── View Distance: High
├── Foliage: Medium
└── Texture: High (2K-4K)

[High/Ultra] — RTX 3070+ / 1440p-4K / 60fps 目标
├── Lumen: ON (Hardware Ray Tracing if available)
├── Nanite: ON
├── Shadows: Ultra, VSM
├── Post-Process: Ultra (SSR/Volumetric Fog/Bloom full)
├── View Distance: Epic
├── Foliage: High
└── Texture: Ultra (4K)

[PS5 Performance] — 1440p 动态 / 60fps
├── Lumen: ON (Software)
├── 等同 PC Medium-High 之间

[PS5 Quality] — 4K / 30fps
├── Lumen: ON (可能 Hardware)
├── 等同 PC High
```

## 四、低配测试实操建议

### 4.1 你应该准备的测试环境

| 测试层级 | 硬件 | 用途 |
|---|---|---|
| **⭐ 最低配测试机** | GTX 1060 6GB + i5-8400 + 8GB RAM | 验证最低配置能否 1080p/30fps |
| **推荐配测试机** | RTX 2060 + i5-10400 + 16GB RAM | 验证推荐配置能否 1080p/60fps |
| **PS5 DevKit/TestKit** | PS5 标准版 | 认证前必须测试 |

> **成本提示：** 如果你没有低配机器，可以在你的开发机上通过以下方式模拟：
> - UE5 Editor 里使用 `r.ScreenPercentage 50` 模拟低分辨率
> - 使用 Scalability 设定为 Low
> - 用 `t.MaxFPS 30` 锁帧测试
> - 但这**不能完全替代**真实低配硬件测试

### 4.2 关键性能指标

| 指标 | 最低配目标 | 推荐配目标 | PS5 目标 |
|---|---|---|---|
| **帧率** | ≥30fps 稳定 | ≥60fps 稳定 | 60fps(Perf) / 30fps(Quality) |
| **帧时间** | ≤33.3ms | ≤16.6ms | 同左 |
| **1% Low** | ≥25fps | ≥50fps | 不低于目标帧率的80% |
| **加载时间** | ≤30秒(HDD) | ≤15秒(SSD) | ≤5秒 |
| **内存占用** | ≤6GB | ≤10GB | ≤12GB |
| **显存占用** | ≤4GB | ≤6GB | 共享内存池 |

### 4.3 性能测试重点场景

对于类魂/恐怖生存游戏，以下场景最容易出性能问题：

| 场景 | 压力来源 | 测试方法 |
|---|---|---|
| **Boss 战** | 多特效 + 多骨骼动画 + AI 计算 | 用最低配置打完整 Boss 战 |
| **大型房间/开阔区域** | 大量 Draw Call + 远景 | 站在能看到最多物体的位置 |
| **多敌人区域** | AI Tick + 骨骼网格体 + 寻路 | 同屏 5-8 个敌人的区域 |
| **光照密集区域** | 多个动态点光源 (蜡烛/火把) | 一个房间放满蜡烛然后战斗 |
| **粒子特效密集** | 火焰/毒雾/魔法效果 | 叠加多个特效时 |
| **快速场景切换** | Level Streaming / 加载 | 开门进入新区域的一瞬间 |

## 五、发行时间线与性能里程碑

| 阶段 | 性能目标 | 说明 |
|---|---|---|
| **白盒阶段** | 不需要关注性能 | 功能优先 |
| **Alpha** | 推荐配跑 30fps 即可 | 确认没有致命性能问题 |
| **Beta** | 推荐配稳定 60fps | 开始低配测试 |
| **RC** | 最低配稳定 30fps + PS5 两种模式通过 | 认证提交前必须达标 |
| **Gold** | 所有配置达标 + 压力测试通过 | 发行 |

## 六、关键建议总结

### ✅ 做这些

1. **最低配置定在 GTX 1060 / i5-8400 / 8GB RAM** — 覆盖 80%+ Steam 用户
2. **目标 1080p + 提供 Low/Medium/High/Ultra 四档** — 让玩家自己选
3. **PS5 提供 Performance/Quality 双模式** — 认证标配
4. **尽早在低配上测试** — 不要等到最后才发现跑不动
5. **Lumen 做为可选** — Low 画质关闭，Medium+ 开启
6. **DualSense 至少支持基础震动** — PS5 认证要求

### ❌ 不要做这些

1. **不要试图支持 GTX 750/HD 4400 级别** — 这些硬件跑不了 UE5 DX12
2. **不要只测 4K 高配** — 你的大部分玩家在 1080p
3. **不要在没有真实低配硬件测试的情况下提交 PS5 认证** — 会被拒
4. **不要在最低配置下也强制 Lumen** — 性能杀手
5. **不要忽视 1% Low** — 平均 60fps 但 1% Low 只有 15fps = 玩家体验极差
