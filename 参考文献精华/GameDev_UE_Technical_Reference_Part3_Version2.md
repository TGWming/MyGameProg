# 十五、IK 基础概念

## 15.1 FK vs IK

| | 正向运动学（FK） | 逆向运动学（IK） |
|---|---|---|
| **方向** | 关节参数 → 末端位置 | 末端目标位置 → 关节参数 |
| **输入** | 用户设定所有关节角度 | 仅设定末端受动器目标 |
| **来源** | 机器人运动学方程 | 机器人运动学方程（反向求解） |
| **应用** | 动画师手动摆关键帧 | 脚部地面适配、手部抓取、门把手交互 |

## 15.2 末端受动器（End Effector）

- 机械手臂抓取位置
- 角色开门时手的位置
- 角色行走时脚的位置
- **不一定在末端**，可以是内部关节（如肘部、膝盖）

# 十六、IK 问题分析

## 16.1 解的情况

IK 问题可能有 **一个解、多个解或无解**。

## 16.2 两个关键步骤

| 步骤 | 说明 |
|---|---|
| **可达性分析** | 目标是否在可达工作空间内？避免搜索不存在的解 |
| **终止条件** | 限制迭代次数 / 末端受动器位移差 < 公差阈值 |

## 16.3 最佳实践

- 开始时骨骼摆出**接近目标的姿势** → 算法聚焦最近解
- 可用解数量取决于目标位置和运动链自由度
- 目标不可达 = **过度约束问题**
- 目标可达但多解 = **冗余问题**（可能有无限解）

# 十七、IK 求解算法全览

## 17.0 四大算法家族

```
IK 求解算法
├── 解析法（Analytic）          ← 简单场景，Two Bone IK
├── 数值法（Numerical）
│   ├── 雅可比矩阵法（Jacobian） ← Full Body IK
│   ├── 牛顿法（Newton）         ← 高精度但高成本
│   └── 启发式法（Heuristic）    ← CCD IK、FABRIK
├── 数据驱动法（Data-driven）    ← Motion Matching 相关
└── 混合法（Hybrid）            ← SIK、统计方法、并行计算
```

---

## 17.1 解析法

### 核心思想
直接用数学公式求解关节角度。

### Two Bone IK（两骨骼 IK）⭐

两段骨骼（不含根骨骼），最简单也最常用。

**解公式**（二维，两杆 l₁, l₂，目标 (x, y)）：

```
θ₁ = cos⁻¹((l₁² + x² + y² - l₂²) / (2·l₁·√(x² + y²)))
θ₂ = cos⁻¹((l₁² + l₂² - (x² + y²)) / (2·l₁·l₂))
```

**三维扩展**：三点确定平面 → 需要增加 **PoleVector（极坐标点）** 来确定旋转平面

**UE4 实现**：
- `SolveBasicTwoBoneIK()`
- `FAnimNode_TwoBoneIK`
- ALS V4 插件中的 FootIK

**局限**：关节数更多时，解析法变得极其复杂

---

## 17.2 数值法 — 雅可比矩阵族

### 核心思想

雅可比矩阵 J 是运动链对关节角度 θ 的偏导数矩阵。
通过迭代调整关节，使末端受动器逐步逼近目标。

```
J(θ)ᵢⱼ = ∂sᵢ/∂θⱼ    （i=末端受动器, j=关节）
误差: e = target - current_position
近似: Δs ≈ J·Δθ
求解: Δθ ≈ J⁻¹·e
```

**问题**：J 可能不是方阵、不可逆、存在奇异性

### 六种变体对比

| 方法 | 公式 | 优点 | 缺点 |
|---|---|---|---|
| **转置法** | Δθ = αJᵀe | 奇异点附近不会未定义 | 收敛慢，运动不稳定 |
| **伪逆法** | Δθ = J†e = Jᵀ(JJᵀ)⁻¹e | 适用所有矩阵 | 奇异点附近抖动大 |
| **DLS** | Δθ = Jᵀ(JJᵀ+λ²I)⁻¹e | 避免奇异性，比伪逆好 | λ选择关键，大λ降低精度 |
| **SVD** | J = UDVᵀ → J† = VD†U† | 提供正交基，处理奇异点 | 计算量较大 |
| **伪逆DLS** | 在DLS下使用SVD | 奇异点附近更平滑 | 计算成本更高 |
| **SDLS** | 每个奇异向量单独调阻尼 | 最优结果，收敛最少迭代 | **计算最慢** |

### 其他雅可比方案

- Gauss-Seidel 迭代法
- "半雅可比矩阵"降低计算成本
- 扩展雅可比矩阵（增加额外行）
- Feedback IK（FIK）：控制论视角，无需矩阵求逆
- 多优先级 DLS 框架：避免多任务冲突
- 增强雅可比 → 优先级 IK（PIK）：先强制重要属性

### 合并约束的方��

| 方法 | 说明 |
|---|---|
| 投影法 | 无约束解投影到可行空间（不保证最优） |
| 惩罚法 | 增加移动限制（常收敛到较差结果） |
| 零空间投影 | 任务优先级 + 雅可比零空间 |
| SNS/IKTC | 机器人领域，放弃越界关节 |
| 加权法 | Δθ = W·J⁻¹·e，W∈[0,1] 加权各关节 |

### UE4/UE5 应用

> **UE4.26 FullBodyIK** 采用**雅可比矩阵法**，基于 ControlRig 插件
> **UE5** 采用全新解算器，基于 **XPBD 刚体模拟**，性能更优

---

## 17.3 数值法 — 牛顿法

- 使用 Hessian 矩阵（二阶导数）
- **不会遇到雅可比的奇异性问题**
- 但：复杂、难实现、每次迭代计算成本高
- 常见于计算机建模工具（非实时游戏）

---

## 17.4 数值法 — 启发式法

> 不用复杂方程，用简单操作迭代逼近。计算成本低，适合实时应用。
> 局限：可能产生非自然动作，不考虑全局约束。

### CCD IK（循环坐标推演）⭐

**原理**：一次调整一个关节，最小化末端受动器到目标的距离

**流程**：
```
从末端受动器开始，向根部方向遍历：
  对每个关节 pᵢ:
    计算 θᵢ = angle(target, pᵢ, end_effector)
    旋转 pᵢ 使 θᵢ → 0
    更新所有子关节位置
重复直到末端受动器足够接近目标
```

**优点**：
- 实现极简单（只用点积 + 叉积）
- 数值稳定，线性时间复杂度
- 容易加入局部角度约束

**缺点**：
- 运动分布不佳（过分强调末端附近关节）
- 可能产生大角度旋转 → 振荡/不连续
- 目标靠近 base 时链条可能自卷
- **只处理串行链**

**扩展**：
- CAA（圆弧对齐算法）
- IIK（归纳 IK + UPM + 学习算法）
- IBK（逆分支运动学）：偏置因子校正旋转

**UE4 实现**：`FAnimNode_CCDIK`

---

### FABRIK（前向后向到达 IK）⭐

**原理**：不使用角度旋转，直接沿直线更新关节位置。前向（末端→根）+ 后向（根→末端）迭代。

**流程**：
```
Forward pass（从末端到根）:
  p'ₙ = target
  for i = n-1 → 1:
    在线段 (pᵢ, p'ᵢ₊₁) 上，距离 p'ᵢ₊₁ 为 dᵢ 处放置 p'ᵢ

Backward pass（从根到末端）:
  p'₁ = original_root_position
  for i = 2 → n:
    在线段 (p'ᵢ₋₁, pᵢ) 上，距离 p'ᵢ₋₁ 为 dᵢ₋₁ 处放置 p'ᵢ

重复直到收敛
```

**优点**：
- 简单、灵活、低计算成本
- 有效处理**闭环**和**多末端受动器**问题
- 即使误差容忍 = 0 也能收敛
- **运动分布到所有关节**（不像 CCD 偏向末端）
- 奇异点附近表现良好

**缺点**：
- 在 position space 而非 orientation space 中求解
- 方向约束下连续性较差
- 小运动链 + 严格约束 → 可能死锁
- 与 CCD 类似，不能整合全局模型约束

**UE4 实现**：`FAnimNode_Fabrik`

---

### CCD vs FABRIK 对比

| 特性 | CCD | FABRIK |
|---|---|---|
| 计算方式 | 角度旋转 | 位置更新 |
| 运动分布 | 偏向末端关节 | **均匀分布** |
| 多末端受动器 | 扩展后支持 | **原生支持** |
| 闭环 | 不直接支持 | **支持** |
| 奇异性 | 较好 | **更好** |
| 运动链增大时性能 | 下降较多 | **变化不大** |
| 自卷问题 | 有 | 无 |
| UE4 节点 | FAnimNode_CCDIK | FAnimNode_Fabrik |

## 17.5 数据驱动法

- 使用预先学习的姿势 → 末端受动器位置匹配可行姿势
- 分两类：**基于误差** / **基于实例**
- 基于实例最常见：预录动作数据库 + 径向基函数混合
- 效率取决于数据库大小
- 缺点：离线训练、姿态不连续性、与训练数据高度相关

> **直接关联**：Motion Matching + Learned Motion Matching（详见第五章）

---

## 17.6 混合法

### 统计方法
- 序贯蒙特卡罗（SMCM）：采样法 + FK 求解 IK，避免逆矩阵
- Particle IK：概率密度函数作为软约束

### 并行计算
- GPU 加速 DLS
- 八叉树存储动捕数据 + 并行过滤

### SIK（连续逆运动学）
- 解析迭代法，实时重建全身运动
- 输入：手腕、脚踝、头部、骨盆位置
- 流程：Spine 配置 → Clavicle 定位 → 四肢 IK
- 使用生物力学约束保证姿势自然

---

## 17.7 生物力学约束

### 两组约束

| 类型 | 示例 |
|---|---|
| 高级控制 | 位置、方向、视觉、平衡 |
| 环境交互 | 地板接触、抓取、碰撞避免 |

### 保证动作自然的方案

| 方案 | 说明 |
|---|---|
| 基于物理的角色动画 | 动量/惯性/力约束 → IK 更自然 |
| 以肌肉为基础 | 肌肉骨骼模型驱动 |
| 数据驱动 | 学习大量动作（耗时但有效） |

### UE5 的方案

> UE5 FullBodyIK 采用**基于 XPBD 的刚体模拟**
> 参考：[UE5 FullBodyIK 学习](https://zhuanlan.zhihu.com/p/381469750)

# 十八、IK 算法选型讨论

## 18.1 类人角色推荐

| 算法族 | 适用性 | 注意事项 |
|---|---|---|
| **雅可比矩阵** | ✅ 高自由度（24+关节） | 抖动问题，DLS/SDLS 较好但慢 |
| **启发式** | ✅ 跟踪多控制点 | 不能整合全局约束，可能不自然 |
| **混合（SIK）** | ✅ 计算高效 + 重建质量好 | 全局限制难以应用 |
| **数据驱动** | ✅ 当前趋势 | 需离线训练，结果依赖数据库 |

## 18.2 非拟人角色推荐

| 场景 | 推荐算法 | 原因 |
|---|---|---|
| 单链（尾巴） | **FABRIK** | 高效、灵活、链增大时性能稳定 |
| 多腿（昆虫） | **FABRIK** | 原生多末端受动器支持 |
| 四足动物 | 雅可比 + 物理 | 生物力学约束重要 |
| 蛇/绳索 | FABRIK / CCD | 简单高效 |
| 复杂生物（龙） | **混合法** | 不同关节用不同处理 |

## 18.3 算法可扩展性

| 算法 | 可扩展性 | 说明 |
|---|---|---|
| 解析法 | ❌ 差 | 模型变化需完全重新设计 |
| 雅可比/牛顿 | ✅ 好 | 容易适应不同链，但时间随 DoF 增加 |
| CCD | ⚠️ 中 | 性能随链增大下降较多 |
| FABRIK | ✅ 很好 | 链增大时计算时间变化不大 |
| 数据驱动 | ❌ 差 | 不同骨架需重新训练 |
| 混合法 | ⚠️ 中 | 需重新定义层次结构 |

## 18.4 未来方向

1. **IK + 动作捕捉**：合成非现实动作、风格迁移
2. **实时骨骼重定向**：不同比例角色间映射（二足→四足）
3. **解剖学/生理学约束**：肌肉骨骼模型、spring/mass 模型
4. **深度学习**：只需机械结构，不需先验信息
5. **肌肉力学**：最小肌肉力变化 → 解决 IK 冗余
