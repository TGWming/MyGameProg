## 2. 视觉需求详细描述

### 2.1 核心视觉元素（按时间顺序）

```
阶段 1：天空变红（0-5s）
   - 天空颜色从原色 → 血红色
   - 太阳变红 / 异化
   - 由 SkyCreator 处理 ✅
   
阶段 2：血肉染色扩散（2-15s）⭐ 核心难点
   - 从某个起点（如玩家上方）开始
   - 像染料/血滴在画布上"晕染"扩散
   - 不是硬切边界，是有机羽化的渐进过程
   - 最终覆盖整个天空
   - 血肉中间隐约能看到人脸轮廓
   
阶段 3：血雨开始（8-30s）
   - 由 SkyCreator 处理 ✅
   
阶段 4：巨大眼睛浮现（24-30s）
   - ��个巨大眼睛从血肉中"破壳"
   - 缓慢浮起到天空表面
   - 眼皮逐渐睁开
   - 玩家能清晰看到几个注视的眼球
```

### 2.2 关键视觉关键词
- **染料晕染**：像水彩在湿宣纸上扩散
- **有机扩散**：不是几何形状，是不规则的血管/真菌生长感
- **渗透感**：边缘有渐变羽化，没有硬边
- **画笔感**：像画家用毛笔逐渐涂满
- **从点到面**：扩散有明确的起点和方向
- **多层次**：血肉 + 人脸轮廓 + 眼睛 = 三个视觉层次

### 2.3 不要的效果
- ❌ 整个天空均匀变化（缺乏扩散感）
- ❌ 硬边切割（缺乏有机感）
- ❌ 一次性突变（缺乏戏剧节奏）
- ❌ 纯 Opacity 0→1 渐变（看起来像幽灵）

## 6. 核心技术原理：Dissolve Shader

### 6.1 为什么不能用 Opacity Lerp

```
❌ 错误做法：
   T=0s:   Opacity = 0 (完全透明)
   T=15s:  Opacity = 1 (完全显现)
   ↓
   结果：整个 Mesh 均匀地从透明变不透明
   视觉：像幽灵慢慢出现，没有"扩散感"
```

### 6.2 Dissolve Shader 原理

```
✅ 正确做法：
   - 准备一张噪点遮罩贴图（Noise Mask）
   - 每个像素的显示与否，由比较"动画阈值 vs 该像素的噪点值"决定
   - 阈值动画 = 染色过程
   
   T=0s:   阈值=0   → 所有像素都看不见
   T=5s:   阈值=0.3 → 噪点值<0.3 的像素显现（少量点出现）
   T=10s:  阈值=0.6 → 噪点值<0.6 的像素显现（大面积扩散）
   T=15s:  阈值=1.0 → 所有像素都显现（完全覆盖）
```

### 6.3 视觉演示��一图理解）

```
噪点遮罩贴图（黑白图）：
┌─────────────────────┐
│ ░ ▒ ▓ █ ▓ ▒ ░ ░     │
│ ▒ ▓ █ █ █ ▓ ▒ ░     │  黑色 = 噪点值低（先出现）
│ ▓ █ █ █ █ █ ▓ ▒     │  白色 = 噪点值高（后出现）
│ ▒ ▓ █ █ █ ▓ ▒ ░     │
│ ░ ▒ ▓ █ ▓ ▒ ░ ░     │
└─────────────────────┘

阈值动画过程:

T=0s   阈值=0.0  → ___________________ (空)
T=3s   阈值=0.2  → ░ ░ ░ ░          (零星点)
T=6s   阈值=0.4  → ░░░ ▒░░ ░░       (开始扩散)
T=9s   阈值=0.6  → ░░░░ ▒▒░░░░ ░░░  (大面积)
T=12s  阈值=0.8  → ▓▓░░░░ ▓▓▓░░░░   (接近全满)
T=15s  阈值=1.0  → ████████████████ (完全覆盖)

视觉效果就像染料在画布上晕开 🎨
```

### 6.4 材质核心算法（伪代码）

```glsl
// === 输入参数 ===
float DissolveAmount = 0.0;     // 0→1 由蓝图驱动
Texture2D NoiseMask;             // 噪点遮罩
Texture2D FleshAlbedo;           // 血肉颜色
float EdgeWidth = 0.05;          // 边缘渐变宽度
float3 EdgeColor = (2.0, 0.0, 0.0); // 边缘发光色（红）

// === 核心算法 ===
float MaskValue = sample(NoiseMask, UV).r;

// ⭐ 关键：用 smoothstep 让边缘有渐变（染料晕开感）
float Opacity = smoothstep(
    DissolveAmount - EdgeWidth,   // 下界
    DissolveAmount + EdgeWidth,   // 上界
    1.0 - MaskValue               // 反转：让低值先显现
);

// === 颜色 + 边缘发光 ===
float3 BaseColor = sample(FleshAlbedo, UV).rgb;

// 边缘发光（让扩散边缘有"血染"感）
float EdgeGlow = 1.0 - smoothstep(
    0.0, 
    EdgeWidth * 2.0, 
    abs(MaskValue - DissolveAmount)
);
float3 EmissiveColor = BaseColor * EdgeGlow * EdgeColor;

// === 输出 ===
Output.BaseColor = BaseColor;
Output.Opacity = Opacity;
Output.Emissive = EmissiveColor;
```

### 6.5 三个关键参数的视觉影响

#### EdgeWidth（边缘宽度）
```
EdgeWidth = 0.0   → 硬边切割（不要）❌
EdgeWidth = 0.05  → 轻微渐变（细腻）⭐ 推荐
EdgeWidth = 0.2   → 强烈晕染（水彩感）
EdgeWidth = 0.5   → 整体半透（雾化）
```

#### NoiseMask 类型
```
均匀噪点    → 看起来像电视雪花（不要）
Perlin 噪点 → 自然有机的扩散 ⭐ 推荐
Voronoi 噪点 → 块状扩散
手绘噪点    → 完全可控的艺术效果（最贵）
血管纹路    → 沿血管生长的感觉（剑风感）⭐ 推荐
```

#### EdgeColor（边缘色）
```
红色边缘 → 染血感 🩸 ⭐ 推荐
紫色边缘 → 异界感 ⚫
橙色边缘 → 烧灼感 🔥
```

## 7. 扩散起点机制：Distance Mask

### 7.1 为什么需要起点

仅用 NoiseMask 会让血肉**全天空均匀同时显现**。  
我们要的是**从一个点（玩家上方）开始扩散**到外围的感觉。

### 7.2 实现原理：Distance Mask + Noise Mask 组合

```glsl
// 输入额外参数
float3 OriginPoint = (PlayerPos + Z_Offset);  // 扩散起点
float MaxRadius = 5000.0;                      // 最大扩散半径

// 计算每个像素到起点的距离
float DistFromOrigin = length(WorldPosition - OriginPoint);

// 距离归一化为 0-1
float DistMask = saturate(DistFromOrigin / MaxRadius);

// ⭐ 把距离遮罩与噪点遮罩混合（双重控制）
float CombinedMask = lerp(DistMask, NoiseValue, 0.5);

// 用 CombinedMask 替代之前的 NoiseValue
float Opacity = smoothstep(
    DissolveAmount - EdgeWidth,
    DissolveAmount + EdgeWidth,
    1.0 - CombinedMask
);
```

### 7.3 效果

```
T=0s:  起点周围一小圈出现血肉（玩家头顶）
T=5s:  血肉以起点为中心向外扩散
T=15s: 扩散覆盖整个天空
```

### 7.4 进阶：多个扩散点

```glsl
// 多个起点
float3 Origins[4] = {Point1, Point2, Point3, Point4};
float MinDist = 99999;
for (int i = 0; i < 4; i++) {
    MinDist = min(MinDist, length(WorldPos - Origins[i]));
}
// 用 MinDist 替代单点距离
```

效果：4 个染料点同时扩散，最终汇合 ✅

## 8. 实现技术选型：为什么用蓝图

### 8.1 决策矩阵

| 系统 | 推荐实现 | 理由 |
|------|---------|------|
| **Dissolve 材质** | 材质编辑器 | 必须，材质本来就是节点 |
| **BP_FleshDome 调度** | 蓝图 | Timeline 驱动天然适配 |
| **EclipseDirector 演出** | 蓝图 | 演出 = 时间线 + 调用 |
| **眼睛生成系统** | 蓝图 | Spawn + 动画播放 |
| **与 SkyCreator 协作** | 蓝图 | SkyCreator 本身就是蓝图 |

### 8.2 不用 C++ 的核心原因

1. **演出类逻辑 = Timeline 主导**：蓝图 Timeline 节点天生为此设计
2. **材质参数驱动 = 蓝图最高效**：3 个节点搞定，C++ 啰嗦
3. **美术友好**：策划/美术能直接调，C++ 碰不了
4. **与 SkyCreator 协作天然适配**：都是蓝图，0 转换成本
5. **演出代码不需要性能极致**：30 秒一次的剧情触发，蓝图够用

### 8.3 黄金法则（项目通用）

```
✅ 用 C++ 写：核心架构、Manager、Subsystem、底层框架（如 Zone ID 系统）
✅ 用蓝图写：演出、特效调度、关卡逻辑、UI、剧情（如本手册）
```

3. **设置材质属性**：
   - Material Domain: **Surface**
   - Blend Mode: **Translucent**（或 Masked，视效果决定）
   - Shading Model: **Unlit**（天空不需要光照）
   - Two Sided: **True**（球体内部看也要渲染）

### 1.4 创建参数
在材质里添加以下 Scalar/Vector Parameters：
- `DissolveAmount` (Scalar, 默认 0.0)
- `EdgeWidth` (Scalar, 默认 0.05)
- `EdgeColor` (Vector, 默认 (2.0, 0.0, 0.0))
- `EdgeGlowIntensity` (Scalar, 默认 3.0)

### 1.5 节点连接（核心逻辑）

```
[Texture Sample: NoiseMask]  →  R 通道  →  [Mask Value]
                                            ↓
                              [1 - MaskValue] (OneMinus 节点)
                                            ↓
[Param: DissolveAmount]  →  ┐
[Param: EdgeWidth]       →  ┤
                            ├─→ [SmoothStep 节点]
                            │      Min = DissolveAmount - EdgeWidth
                            │      Max = DissolveAmount + EdgeWidth
                            │      Value = (1 - MaskValue)
                            ↓
                       [Opacity Output]

边缘发光：
[Param: DissolveAmount]  ─┐
[MaskValue]              ─┤  →  [Abs(Mask - Dissolve)]  →  [SmoothStep]
                                                              ↓
                                                       [1 - SmoothStep]
                                                              ↓
                                                       [Multiply EdgeGlow]
                                                              ↓
                              [Param: EdgeColor] →  [Multiply]
                                                              ↓
                                                    [Emissive Output]

颜色：
[Texture Sample: FleshAlbedo]  →  [BaseColor / Emissive Output]
```

### 1.6 创建材质实例
- 右键 M_FleshDissolve → Create Material Instance
- 命名为 `MI_FleshDissolve`
- 后续 BP_FleshDome 引用这个 MI

### 1.7 验证
- 创建一个 Sphere Mesh，应用 MI_FleshDissolve
- 在 MI 的 Details 面板手动拖动 DissolveAmount 0 → 1
- 观察是否有"染料晕染"的扩散感
- 调整 EdgeWidth 看边缘羽化效果

### ✅ 步骤 1 完成标准
- 拖动 DissolveAmount 能看到血肉从无到有逐渐覆盖球体
- 边缘有红色发光晕染感
- 不是均匀渐变，而是有"扩散点"的有机感

## 步骤 3：扩散起点机制集成

### 3.1 目标
让 Dissolve 不再是均匀的，而是从某个起点（玩家上方）扩散开。

### 3.2 修改材质 M_FleshDissolve

#### 添加新参数
- `OriginPoint` (Vector Parameter, 默认 (0, 0, 5000))
- `MaxRadius` (Scalar Parameter, 默认 5000.0)
- `DistanceMaskWeight` (Scalar Parameter, 默认 0.5)

#### 修改节点逻辑

```
新增计算 DistMask：
[Absolute World Position]  →  ┐
[Param: OriginPoint]       →  ├─→ [Subtract] → [Length] → [Divide MaxRadius] → [Saturate]
                                                                                      ↓
                                                                              [DistMask 0-1]

混合 NoiseMask 和 DistMask：
[NoiseMask Value]    ─┐
[DistMask]           ─┤  →  [Lerp] (Alpha = DistanceMaskWeight)  →  [CombinedMask]
                                                                          ↓
                                              用 CombinedMask 替代之前的 (1 - MaskValue)
                                                                          ↓
                                                              [SmoothStep] (与之前相同)
```

### 3.3 修改 BP_FleshDome

#### 在 Tick 中更新 OriginPoint

```
Event Tick
   ↓
[Get Player Pawn]  →  [Get Actor Location]
   ↓
[Add Vector] (Z = +5000)  → 玩家上方 5000 单位
   ↓
[Set Vector Parameter Value] (DynamicMaterial, OriginPoint, NewLocation)
```

> 💡 也可以一次性在 PlayDissolve 时设置（如果不需要追踪玩家移动）

### 3.4 验证
- 调用 PlayDissolve
- 观察血肉是否从玩家上方开始扩散
- 玩家移动时扩散点是否跟随（如果开了 Tick）

### ✅ 步骤 3 完成标准
- 血肉从玩家头顶上方一个点开始扩散
- 扩散过程像染料晕开
- DistanceMaskWeight 可调，控制扩散方向感强弱

## 步骤 4：人脸/血肉变体 Mesh

### 4.1 目标
让铺满后的天空不是单调的血肉，能看到人脸轮廓。

### 4.2 实现方式（择一）

#### 方式 A：法线贴图浮雕（推荐）
- 美术做一张人脸 Normal Map
- 应用到 M_FleshDissolve 的 Normal 输入

- 关卡里放几个 BP_FaceMeshSpawner 标记位置
- EclipseDirector 在血肉铺到一定程度时 Spawn 这些 Mesh
- 每个 Mesh 自带 Dissolve 出生动画

### 4.3 推荐方案：A + C 组合
- 主穹顶用方案 A 做基础人脸感
- 关键位置用方案 C 放置突出的大型人脸 Mesh

### ✅ 步骤 4 完成标准

## 步骤 6：演出导演 BP_EclipseDirector

### 6.1 目标
统一调度所有演出元素。

### 6.2 创建 BP_EclipseDirector

1. 新建 Blueprint Class，父类 Actor，命名 `BP_EclipseDirector`
2. **添加变量**（EditInstanceOnly，让 Details 面板可配置）：
   - `TargetSkyCreator` (Actor Reference)
   - `TargetFleshDome` (BP_FleshDome Reference)
   - `EyeSpawnPoints` (Array of Transform)
   - `GiantEyeClass` (Class Reference, 默认 BP_GiantEye)
   - `TotalDuration` (Float, 默认 30.0)

### 6.3 主函数：PlayEclipse

```
Function: PlayEclipse
   ↓
   1. 启动主 Timeline (MasterTimeline, Length = TotalDuration)
   2. Timeline 内��过事件触发各阶段
```

### 6.4 MasterTimeline 配置

添加 Event Track（不用 Float），关键事件：
- Time 0.0: Event_StartSky
- Time 2.0: Event_StartFlesh
- Time 8.0: Event_StartRain
- Time 10.0: Event_StartLighting
- Time 24.0: Event_SpawnEyes
- Time 30.0: Event_Complete

### 6.5 各事件实现

```
Event_StartSky:
   → TargetSkyCreator.SetSkyColor(BloodRed) // 调 SkyCreator 蓝图函数

Event_StartFlesh:
   → TargetFleshDome.PlayDissolve(15.0)

Event_StartRain:
   → TargetSkyCreator.SetRainAmount(1.0)

Event_StartLighting:
   → 调用全局光照变化（步骤 8）

Event_SpawnEyes:
   → For Each EyeSpawnPoints:
        SpawnActor (Class = GiantEyeClass, Transform = SpawnPoint)
        SpawnedEye.PlayBirthAnimation()

Event_Complete:
   → Print "Eclipse Complete"
   → 可选：广播事件给其他系统
```

### 6.6 验证
- 场景拖入 BP_EclipseDirector
- Details 配置好 TargetSkyCreator / TargetFleshDome / EyeSpawnPoints
- BeginPlay 时调用 PlayEclipse() 测试整个流程

### ✅ 步骤 6 完成标准
