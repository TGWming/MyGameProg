## 2. Prompt 类型与格式约定

### 2.1 两种 Prompt 类型

| 类型 | 用途 | 是否改代码 |
|------|------|-----------|
| **诊断类（Diagnostic）** | 了解项目结构、确认介入点、调研现状 | ❌ 不改代码 |
| **执行类（Execution）** | 落地具体代码实现 | ✅ 改代码 |

### 2.2 格式约定
- 所有 Prompt 必须包含在**单一 Code Block 内**（使用 ```` ```text ```` 或 ```` ```prompt ````）
- 便于用户**一键复制**
- 开头必须包含：任务类型、目标、约束
- 结尾必须包含：分步执行要求、汇报格式

## 3. Agent 强制规则

### 3.1 ⚠️ 搜索方式约束

- **🚫 禁止使用 PowerShell** 进行文件搜索或内容查询
  - 原因：在大型项目中 PowerShell 搜索容易导致编辑器/IDE 卡顿
- **✅ 推荐工具**：
  - IDE 内建搜索（Find in Files）
  - `grep`（Git Bash 环境）
  - `findstr`（cmd 环境）
  - 代码索引工具（如 ReSharper、VS 内建索引）

### 3.2 🧱 任务原子化原则

- **每次执行任务必须分步骤**，单步改动范围最小化
- **严禁"一次性大重构"**
- **每完成一个子任务必须简短汇报**：
  - 文件路径
  - 改动摘要（1-2 句）
  - 下一步建议（可选）
- 报告格式：
  ```
  [已完成] 子任务编号 X.Y
  - 文件：path/to/file.cpp
  - 改动：（简短描述）
  - 下一步建议：（如有）
  ```

### 3.3 🛑 编译职责（最重要！）

- **Agent 严禁自行执行编译**
- **编译由用户手动进行**
- 如果编译报错：
  1. 用户将错误信息反馈给 Copilot Chat
  2. Copilot Chat 分析并产出修复 Prompt
  3. Agent 根据修复 Prompt 执行
  4. **Agent 不得在编译失败后自行猜测并修改代码**
- 原因：Agent 自行修复编译错误容易导致代码越改越乱、引入隐藏 Bug

### 3.4 📁 文件改动安全规则

- **改动前必须先读取文件**，确认现有内容
- **不得删除/重命名**未在任务范围内的文件
- **不得修改以下文件**（除非 Prompt 明确要求）：
  - `.uproject`
  - `*.Build.cs`
  - `*.Target.cs`
  - `DefaultEngine.ini` / `DefaultGame.ini`
  - 任何 `.uasset`（资产文件）
- **不得修改 git 配置或执行**：
  - `git push`
  - `git merge`
  - `git rebase`
  - `git reset --hard`
  - 创建/删除分支

### 3.5 🚫 禁止行为清单

- ❌ 自行安装第三方插件、NuGet 包或依赖
- ❌ 修改引擎源码
- ❌ 创建超出任务范围的新文件
- ❌ 在没有明确指令的情况下重构现有代码
- ❌ 修改命名空间、类名、接口签名（除非 Prompt 明确要求）
- ❌ 自行执行单元测试或集成测试（除非 Prompt 要求）
- ❌ 提交代码到远程仓库

### 3.6 ✅ 鼓励行为

- ✅ 改动前读取文件确认上下文
- ✅ 遇到不确定时**停下来询问**而非猜测
- ✅ 改动后输出 diff 摘要
- ✅ 如发现 Prompt 与现有代码冲突，**立即停止并报告**
- ✅ 保留原有代码注释与格式风格

## 4. Copilot Chat 职责

- 与用户进行需求讨论与方案设计
- 输出**结构化、原子化、可一键复制**的 Prompt
- Prompt 必须包含明确的：
  - 任务类型（诊断 / 执行）
  - 目标与范围
  - 约束条件（含本文档所有约束）
  - 分步骤清单
  - 汇报要求
- 接收用户反馈的 Agent 报告或编译错误
- 根据反馈迭代产出下一个 Prompt
- **不得直接执行代码**（Copilot Chat 是讨论端，不是执行端）

---

## 5. 用户职责

- 提出明确的需求或问题
- 与 Copilot Chat 充分讨论后确认方案
- 复制 Copilot Chat 产出的 Prompt 给 Agent 执行
- **手动编译项目**
- 将 Agent 的执行报告反馈给 Copilot Chat
- 将编译错误（如有）反馈给 Copilot Chat
- 对 Agent 的改动进行 Code Review

## 7. 异常处理流程

### 7.1 Agent 遇到不确定情况
1. **立即停止当前任务**
2. 输出报告：
   - 当前进度（已完成哪些子任务）
   - 阻塞原因（具体描述）
   - 建议方案（如有）
3. **等待用户/Chat 进一步指令**
4. **不得自行猜测继续执行**

### 7.2 编译失败
1. 用户将完整编译错误日志反馈给 Copilot Chat
2. Copilot Chat 分析错误并产出修复 Prompt
3. 用户复制 Prompt 给 Agent 执行
4. Agent 完成修复后用户重新手动编译
5. 循环直到编译通过

### 7.3 Agent 改动超出范围
1. 用户立即停止 Agent
2. 用 git diff 检查改动范围
3. 回滚超出范围的改动
4. 反馈给 Copilot Chat 调整 Prompt 措辞

### 7.4 Prompt 与现有代码冲突
1. Agent 发现冲突立即停止
2. 输出冲突报告：
   - 冲突文件
   - 冲突点描述
   - 当前代码与 Prompt 期望的差异
3. 等待 Copilot Chat 调整方案
