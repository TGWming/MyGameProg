⏱️ Cooldown 定时器精确语义
核心思路一句话
Cooldown 计时器只是一个"延迟解析"的提醒；它到期时，重新看一眼当前栈顶应该是什么，再决定切不切。

完整时序举例
假设：A 区 Cooldown=0s，B 区 Cooldown=1.5s

场景 1：玩家进 B 后停留
Code
t=0.0s  玩家进 B → 应用 B 配置 → 启动 1.5s 定时器
t=1.5s  定时器到期 → 检查当前栈顶 = B → 还是 B，无需切换 ✅
场景 2：玩家进 B 后 0.5s 回到 A（你的核心需求）
Code
t=0.0s  玩家进 B → 应用 B 配置 → 启动 1.5s 定时器
t=0.5s  玩家回 A → 栈更新（栈顶=A），但 cooldown 中，不立刻应用 ⏸️
t=1.5s  定时器到期 → 检查当前栈顶 = A → 切换到 A 配置 ✅
场景 3：玩家在 cooldown 期间反复横跳
Code
t=0.0s  进 B → 应用 B → 启动 1.5s 定时器
t=0.3s  回 A → 栈顶=A，cooldown 中，不切 ⏸️
t=0.6s  又进 B → 栈顶=B，cooldown 中，不切 ⏸️
t=0.9s  又回 A → 栈顶=A，cooldown 中，不切 ⏸️
t=1.5s  定时器到期 → 检查当前栈顶 = A → 切换到 A 配置 ✅
结果：玩家边界横跳期间 B 一直显示，1.5s 后才切到 A —— 完美防抖

场景 4：定时器到期那一刻，新目标自己也有 cooldown
Code
t=0.0s  进 B（Cooldown=1.5s）→ 应用 B
t=0.5s  回 A（Cooldown=0s）→ 等待
t=1.5s  定时器到期 → 切到 A → 因为 A 的 Cooldown=0，立刻可再次响应切换
场景 5：cooldown 结束后栈空了
Code
t=0.0s  进 B → 应用 B → 启动 1.5s 定时器
t=0.5s  完全离开 B → 栈空，cooldown 中不应用 ⏸️
t=1.5s  定时器到期 → 栈空 → 应用 DefaultZoneID 配置 ✅
🔑 关键机制要点（这些会写进 Prompt）
机制	行为
进入新区域时	立刻应用配置（如果不在 cooldown 中），并启动该配置的 cooldown 定时器
Cooldown 期间 EnterZone/ExitZone	照常更新栈（栈是真实状态），但不调用 ApplyConfig，只记录"待解析"
Cooldown 定时器到期	重新调用 ResolveActiveZoneID() 拿当前栈顶（或 DefaultZoneID），应用对应配置
新切换的区域如果 Cooldown>0	启动新的 cooldown 定时器，覆盖旧的
Cooldown=0 的配置	等同于"无防抖"，每次进就立刻切
BlendTime vs Cooldown	两个独立机制：Cooldown 控制"何时开始切"，BlendTime 控制"切的时候过渡多久"。可以并存，互不冲突。
一个值得一提的边界 case：定时器到期后的"二次 cooldown"
t=1.5s 定时器到期，发现要切到 A，A 的 Cooldown=0 → 立刻切，无新定时器
t=1.5s 定时器到期，发现要切到 A，A 的 Cooldown=2s → 立刻切，启动 A 的 2s 新定时器

也就是说每次"实际应用配置"后都会基于该配置的 Cooldown 重置定时器，这样语义最一致。

==================== 设计依据（必读） ====================
- 类继承：AActor，Blueprintable，关卡里只放一个实例
- ZoneID 类型：FName（与音频系统统一，便于未来联动）
- Manager API 与音频系统对齐：EnterZone(FName, int32, int32) / ExitZone(FName, int32, int32)
- 栈：TArray<FActiveEnvZoneEntry>（含 ZoneID/Priority/TriggerUniqueID），独立声明不复用音频
- ResolveActiveZoneID：栈空→DefaultZoneID；否则取栈中最高 Priority（同 Priority 取最近 Push）
- BlendTime：Tick 插值控制
- Cooldown 语义（重要！）：
  * 应用配置后启动 FTimerHandle（CooldownSeconds 秒）
  * Cooldown 期间 EnterZone/ExitZone 照常更新栈，但暂不调用 ApplyConfig
  * 定时器到期 → 重新 ResolveActiveZoneID → 应用结果配置（这次再启动新 cooldown）
  * BeginPlay 时初次应用 DefaultZoneID 配置不启动 cooldown
  * Cooldown=0 视为无防抖，立刻应用且不启动定时器
- SkyCreator 调用：反射 FindFunction(FName("SetRainAmount")) + ProcessEvent，传一个 float
- 场景引用类型：
  * TargetFog: AExponentialHeightFog*
  * TargetGlobalLight: APointLight*
  * TargetSkyCreator: AActor*（蓝图中介，不强类型依赖 SkyCreator 插件）
- 玩家过滤已在 Volume 端做过（Cast<ACharacter>+IsPlayerControlled），Manager 端无需重复
- ValidateReferences 是 CallInEditor 按钮，输出 UE_LOG Warning 列出未配置项
- BlendTime 期间应用新配置：从"当前实际值"插到"目标配置值"，避免硬切；
  若处于上次 Blend 中途又有新切换，则把"当前插值中间值"作为新起点

    class SOUL_API AEnvironmentZoneManager : public AActor
    {
        GENERATED_BODY()
    public:
        AEnvironmentZoneManager();

        // ====== 场景引用（Details 下拉选）======
        UPROPERTY(EditInstanceOnly, BlueprintReadWrite, Category="Scene References",
            meta=(DisplayName="高度雾对象"))
        AExponentialHeightFog* TargetFog;

        UPROPERTY(EditInstanceOnly, BlueprintReadWrite, Category="Scene References",
            meta=(DisplayName="质感灯对象"))
        APointLight* TargetGlobalLight;

        UPROPERTY(EditInstanceOnly, BlueprintReadWrite, Category="Scene References",
            meta=(DisplayName="天空创建器中介Actor（含SetRainAmount蓝图函数）"))

        UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="Zone Configurations",
            meta=(DisplayName="区域配置表"))
        TMap<FName, FZoneEnvConfig> ZoneConfigs;

        UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="Zone Configurations",
            meta=(DisplayName="默认区域 ID（栈空时使用）"))

        // ====== 来自 Volume 的 API（与音频系统签名一致）======

        void ExitZone(FName InZoneID, int32 InPriority, int32 InTriggerUniqueID);

        // ====== 编辑器辅助 ======
        UFUNCTION(CallInEditor, Category="Validation",
            meta=(DisplayName="检查场景引用与配置完整性"))
        void ValidateReferences();

    protected:
        virtual void BeginPlay() override;
        virtual void Tick(float DeltaTime) override;

        // 从栈解析当前应用 ZoneID（栈空→DefaultZoneID；否则取最高 Priority，同优先级取最近 Push）
        FName ResolveActiveZoneID() const;

        // 应用一个配置（启动 BlendTime 插值；如 Cooldown>0 则启动 cooldown 定时器）
        void ApplyConfig(FName ZoneID, const FZoneEnvConfig& Config);

        // Cooldown 定时器到期回调：重新解析并应用
        UFUNCTION()
        void OnCooldownExpired();

        // 内部：尝试解析并应用（受 cooldown 限制）；若处于 cooldown 仅记录"待解析"
        void RequestResolve();

        // 实际写入 Fog / Light / SkyCreator 当前配置（每帧 Tick 调用，使用 Blend 中间值）
        void ApplyCurrentBlendedToScene();

    private:
        UPROPERTY(Transient)
        TArray<FActiveEnvZoneEntry> ActiveZoneStack;

        UPROPERTY(Transient)
        FName CurrentAppliedZoneID = NAME_None;

        FZoneEnvConfig BlendFromConfig;
        // Blend 终点
        FZoneEnvConfig BlendToConfig;
        // Blend 进度 0~1
        float BlendAlpha = 1.0f;
        // 此次 Blend 的总时长（来自目标配置 BlendTime）
        float CurrentBlendDuration = 0.f;

        // Cooldown
        FTimerHandle CooldownTimerHandle;
        bool bInCooldown = false;
    };

【步骤 4】实现栈管理 + EnterZone / ExitZone / ResolveActiveZoneID / RequestResolve / OnCooldownExpired

- EnterZone(InZoneID, InPriority, InTriggerUniqueID)：
    - 栈尾追加 FActiveEnvZoneEntry{InZoneID, InPriority, InTriggerUniqueID}
    - 调用 RequestResolve()
    - UE_LOG 区域进入 + 当前栈大小

- ExitZone(InZoneID, InPriority, InTriggerUniqueID)：
    - 在栈中查找匹配（ZoneID + Priority + TriggerUniqueID 三者全匹配的第一个）→ RemoveAt
    - 若找不到匹配项：UE_LOG Warning 并返回（防御）
    - 调用 RequestResolve()

- ResolveActiveZoneID() const：
    - 若 ActiveZoneStack 为空 → 返回 DefaultZoneID
    - 否则：遍历找最高 Priority 的；如有同优先级，取索引最大者（最近 Push）
    - 返回该项的 ZoneID

- RequestResolve()：
    - FName Target = ResolveActiveZoneID()
    - 若 Target == CurrentAppliedZoneID → 无需切换，直接返回
    - 若 bInCooldown == true → UE_LOG Verbose("Cooldown 中，暂缓切换到 %s")，返回
    - 否则查 ZoneConfigs.Find(Target)：
        * 找到 → ApplyConfig(Target, *Cfg)
        * 找不到 → UE_LOG Warning("ZoneID=%s 在 ZoneConfigs 中未配置")，使用空 FZoneEnvConfig{} + 不启动 cooldown

- OnCooldownExpired()：
    - bInCooldown = false
    - UE_LOG Verbose("Cooldown 到期")
    - RequestResolve()  // 重新解析，可能切到新栈顶

- 完成后汇报：四个函数实现要点

【步骤 5】实现 ApplyConfig / Tick 插值 / ApplyCurrentBlendedToScene / SkyCreator 反射调用

- ApplyConfig(ZoneID, Config)：
    - 若当前正在 Blend（BlendAlpha < 1.0f）：
        * 把"当前插值中间结果"作为新的 BlendFromConfig（按 BlendAlpha 在 BlendFromConfig 与 BlendToConfig 之间插值出当前快照）
    - 否则 BlendFromConfig = BlendToConfig（保持上次终点为新起点）
    - BlendToConfig = Config
    - CurrentBlendDuration = FMath::Max(Config.BlendTime, 0.0001f)
    - BlendAlpha = 0.0f
    - CurrentAppliedZoneID = ZoneID
    - 启动 cooldown：
        * 若 Config.CooldownSeconds > 0：
            bInCooldown = true
            GetWorld()->GetTimerManager().SetTimer(CooldownTimerHandle, this,
                &AEnvironmentZoneManager::OnCooldownExpired, Config.CooldownSeconds, false);
        * 否则 bInCooldown = false（保险起见 ClearTimer）
    - UE_LOG Log("应用配置: %s, BlendTime=%.2f, Cooldown=%.2f")

- Tick(DeltaTime)：
    - Super::Tick(DeltaTime)
    - 若 BlendAlpha < 1.0f：
        BlendAlpha = FMath::Clamp(BlendAlpha + DeltaTime / CurrentBlendDuration, 0.f, 1.f);
    - 始终调用 ApplyCurrentBlendedToScene()（即使 BlendAlpha=1 也应用一次终值，保证场景同步）
      * 性能优化：可加一个 bDirty 标记，仅在 BlendAlpha 变化时调用；先锋阶段直接每帧调用即可

- ApplyCurrentBlendedToScene()：
    - 用 FMath::Lerp 计算三个标量当前值：
        FogDensityNow = Lerp(BlendFromConfig.FogDensity, BlendToConfig.FogDensity, BlendAlpha)
        AttenuationNow = Lerp(BlendFromConfig.GlobalLightAttenuationRadius, BlendToConfig.GlobalLightAttenuationRadius, BlendAlpha)
        RainNow = Lerp(BlendFromConfig.SkyCreatorRainAmount, BlendToConfig.SkyCreatorRainAmount, BlendAlpha)
    - 应用到场景：
        * if (TargetFog && TargetFog->GetComponent()) TargetFog->GetComponent()->FogDensity = FogDensityNow;
          ⚠️ 4.27 中 GetComponent() 返回类型注意，必要时用 TargetFog->FindComponentByClass<UExponentialHeightFogComponent>()，
          或直接 TargetFog->GetComponent()（UExponentialHeightFogComponent*，AExponentialHeightFog 自带 GetComponent）。
          请 Agent 先打开 Engine/ExponentialHeightFog.h 确认正确访问方式后再写。
        * if (TargetGlobalLight && TargetGlobalLight->PointLightComponent)
              TargetGlobalLight->PointLightComponent->SetAttenuationRadius(AttenuationNow);
          （PointLightComponent 是 APointLight 的公开成员）
        * if (TargetSkyCreator) 反射调用 SetRainAmount(float)：
            UFunction* Fn = TargetSkyCreator->FindFunction(FName(TEXT("SetRainAmount")));
            if (Fn)
            {
                struct FParam { float Amount; };
                FParam P{ RainNow };
                TargetSkyCreator->ProcessEvent(Fn, &P);
            }
            else 一次性 Warning（可用 static bool bWarned 防刷屏）
              UE_LOG(LogTemp, Warning, TEXT("[EnvZoneMgr] TargetSkyCreator 未实现 SetRainAmount(float) 蓝图函数"));
