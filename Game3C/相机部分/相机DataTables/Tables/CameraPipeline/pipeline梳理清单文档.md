三、各阶段详细说明

Stage 1: Input Context Gather（输入上下文收集）

┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 1: INPUT CONTEXT GATHER                                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  职责：收集当前帧所有相机计算需要的输入数据                                  │
│                                                                              │
│  收集的数据类别：                                                            │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  1.  时间数据                                                                 │
│     · DeltaTime (帧间隔)                                                     │
│     · TotalTime (累计时间)                                                   │
│     · TimeScale (时间缩放，用于慢动作)                                       │
│                                                                              │
│  2.  角色数据                                                                 │
│     · CharacterLocation (角色位置)                                           │
│     · CharacterRotation (角色朝向)                                           │
│     · CharacterVelocity (角色速度)                                           │
│     · CharacterMesh (骨骼网格引用，用于Socket偏移)                           │
│                                                                              │
│  3. 玩家输入数据                                                             │
│     · PlayerYawInput (水平视角输入)                                          │
│     · PlayerPitchInput (垂直视角输入)                                        │
│     · RightStickX/Y (手柄右摇杆)                                             │
│     · TimeSinceLastInput (上次输入后的时间，用于自动回中)                    │
│                                                                              │
│  4. 锁定目标数据                                                             │
│     · LockOnTarget (锁定的Actor)                                             │
│     · LockOnTargetLocation (目标位置)                                        │
│     · TargetSize (目标体型)                                                  │
│     · bHasLockTarget (是否有锁定目标)                                        │
│                                                                              │
│  5. 战斗状态数据                                                             │
│     · bInCombat (是否在战斗中)                                               │
│     · bIsAttacking (是否正在攻击)                                            │
│     · bIsDodging (是否正在闪避)                                              │
│     · bIsParrying (是否正在弹反)                                             │
│     · bExecutionTriggered (是否触发处决)                                     │
│                                                                              │
│  6. 角色状态数据                                                             │
│     · bIsSprinting (是否冲刺)                                                │
│     · bIsCrouching (是否蹲伏)                                                │
│     · bIsInAir (是否在空中)                                                  │
│     · bIsMounting (是否骑乘)                                                 │
│     · bPlayerDead (是否死亡)                                                 │
│                                                                              │
│  7. 环境数据                                                                 │
│     · bInTightSpace (是否在狭小空间)                                         │
│     · EnvironmentDistanceLimit (环境距离限制)                                │
│     · CurrentAreaType (当前区域类型)                                         │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

Stage 2: State Machine Update（状态机更新）

┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 2: STATE MACHINE UPDATE                                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  职责：决定当前帧应该使用哪个相机状态，管理状态转换                          │
│                                                                              │
│  执行步骤：                                                                  │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  Step 2.1: 评估期望状态                                                      │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │  根据InputContext评估当前应该处于什么状态                            │    │
│  │  · 检查死亡 → Death状态                                              │    │
│  │  · 检查处决触发 → Execution状态                                      │    │
│  │  · 检查Boss战 → Boss_Medium/Large/Huge状态                          │    │
│  │  · 检查锁定 → LockOn状态                                             │    │
│  │  · 检查战斗 → Combat状态                                             │    │
│  │  · 检查骑乘 → Mount状态                                              │    │
│  │  · 检查冲刺 → Sprint状态                                             │    │
│  │  · 默认 → Explore状态                                                │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  Step 2.2: 检查转换条件                                                      │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │  · 当前状态是否允许转换到期望状态？                                  │    │
│  │  · 是否满足转换的前置条件？                                          │    │
│  │  · 是否有更高优先级的状态请求？                                      │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  Step 2.3: 开始/更新转换                                                     │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │  如果需要转换：                                                      │    │
│  │  · 记录PreviousState                                                 │    │
│  │  · 设置BlendDuration（根据转换规则）                                 │    │
│  │  · 重置BlendElapsed                                                  │    │
│  │  · 选择BlendCurve                                                    │    │
│  │                                                                       │    │
│  │  如果正在转换中：                                                    │    │
│  │  · 更新BlendElapsed += DeltaTime                                     │    │
│  │  · 计算BlendAlpha = BlendCurve.Evaluate(BlendElapsed/BlendDuration) │    │
│  │  · 检查是否转换完成                                                  │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  输出：                                                                      │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · CurrentState (当前状态)                                                   │
│  · PreviousState (上一个状态)                                                │
│  · BlendAlpha (混合进度 0~1)                                                 │
│  · bIsTransitioning (是否正在转换中)                                         │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

Stage 3: Module Compute Stack（模块计算栈）

┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 3: MODULE COMPUTE STACK                                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  职责：遍历并执行所有激活的计算模块，收集各模块的原始输出                    │
│                                                                              │
│  模块类别：                                                                  │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐              │
│  │ Position Modules│  │Rotation Modules │  │ Distance Modules│              │
│  │ ─────────────── │  │ ─────────────── │  │ ─────────────── │              │
│  │ · FollowTarget  │  │ · PlayerInput   │  │ · BaseDistance  │              │
│  │ · Orbit         │  │ · LookAt        │  │ · TargetSize    │              │
│  │ · Predictive    │  │ · AutoOrient    │  │ · Environment   │              │
│  │ · ScreenOffset  │  │ · LockOnRotation│  │ · Combat        │              │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘              │
│                                                                              │
│  ┌─────────────────┐  ┌─────────────────┐                                   │
│  │   FOV Modules   │  │PostProcess Mods │                                   │
│  │ ─────────────── │  │ ─────────────── │                                   │
│  │ · SpeedFOV      │  │ · Smoothing     │                                   │
│  │ · CombatFOV     │  │ · Shake         │                                   │
│  │ · AimFOV        │  │ · Impact        │                                   │
│  │ · BossFOV       │  │ · DepthOfField  │                                   │
│  └─────────────────┘  └─────────────────┘                                   │
│                                                                              │
│  执行流程：                                                                  │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  FOR each ModuleCategory in [Position, Rotation, Distance, FOV, PostProcess]│
│      FOR each Module in Category                                             │
│          IF Module. IsActiveInState(CurrentState)                             │
│              Output = Module.Compute(DeltaTime, Context, PreviousOutput)    │
│              IF Output. bIsValid                                              │
│                  Add to ModuleOutputCache with (Priority, BlendPolicy)      │
│                                                                              │
│  输出：                                                                      │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · TArray<FModuleOutputEntry> - 所有模块的输出及其元数据                     │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

Stage 4: Modifier Apply（修改器应用）

┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 4: MODIFIER APPLY                                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  职责：更新并应用所有激活的临时修改器效果                                    │
│                                                                              │
│  与Module的区别：                                                            │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  Module（模块）                    Modifier（修改器）                        │
│  · 持续运行                        · 事件触发                                │
│  · 状态驱动激活                    · 有限持续时间                            │
│  · 计算基础参数                    · 叠加临时效果                            │
│  · 例：跟随、锁定旋转              · 例：震动、处决相机                      │
│                                                                              │
│  执行流程：                                                                  │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  1. 更新修改器生命周期                                                       │
│     FOR each Modifier in ActiveModifiers (reverse order)                    │
│         bStillActive = Modifier. Update(DeltaTime)                           │
│         IF NOT bStillActive                                                  │
│             Modifier. Deactivate()                                            │
│             Remove from ActiveModifiers                                      │
│                                                                              │
│  2.  按优先级排序                                                             │
│     Sort ActiveModifiers by Priority                                         │
│                                                                              │
│  3.  应用修改器效果                                                           │
│     FOR each Modifier in ActiveModifiers                                    │
│         Modifier.ModifyOutput(CurrentOutput)                                │
│                                                                              │
│  修改器生命周期：                                                            │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  ┌────────┐   ┌────────┐   ┌────────┐   ┌────────┐   ┌────────┐            │
│  │Inactive│──▶│BlendIn │──▶│ Active │──▶│BlendOut│──▶│Inactive│            │
│  └────────┘   └────────┘   └────────┘   └────────┘   └────────┘            │
│       ▲           │            │            │                                │
│       │           │ Weight     │ Weight     │ Weight                         │
│       │           │ 0→1        │ = 1        │ 1→0                            │
│       └───────────┴────────────┴────────────┘                                │
│                                                                              │
│  可跳过条件：无激活的修改器时跳过此阶段                                      │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

Stage 5: Blend And Solve Conflicts（混合与冲突解决）

┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 5: BLEND AND SOLVE CONFLICTS                                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  职责：合并所有模块输出，解决冲突，生成统一的相机参数                        │
│                                                                              │
│  混合策略类型：                                                              │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  ┌──────────────┬────────────────────────────────────────────────────────┐  │
│  │ 策略         │ 说明                                                    │  │
│  ├──────────────┼────────────────────────────────────────────────────────┤  │
│  │ Additive     │ 加法叠加（震动偏移、位置偏移）                          │  │
│  │ Override     │ 完全覆盖（过场镜头、处决相机）                          │  │
│  │Multiplicative│ 乘法叠加（距离系数）                                    │  │
│  │ Maximum      │ 取最大值（距离下限保护）                                │  │
│  │ Minimum      │ 取最小值（距离上限保护）                                │  │
│  │ Blend        │ 权重混合（状态切换过渡）                                │  │
│  └──────────────┴────────────────────────────────────────────────────────┘  │
│                                                                              │
│  处理流程：                                                                  │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  Step 5.1: 按优先级排序模块输出                                              │
│                                                                              │
│  Step 5.2: 分类处理                                                          │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │ Position处理：                                                       │    │
│  │ · 累加所有Additive偏移                                               │    │
│  │ · 如有Override，使用Override值                                       │    │
│  │                                                                       │    │
│  │ Rotation处理：                                                        │    │
│  │ · 累加所有Additive偏移                                               │    │
│  │ · 如有Override，使用Override值                                       │    │
│  │                                                                       │    │
│  │ Distance处理：                                                        │    │
│  │ · 基础距离 × 所有Multiplicative系数                                  │    │
│  │ · + 所有Additive偏移                                                 │    │
│  │ · 应用Maximum/Minimum限制                                            │    │
│  │                                                                       │    │
│  │ FOV处理：                                                             │    │
│  │ · 基础FOV + 所有Additive偏移                                         │    │
│  │ · 应用全局Min/Max限制                                                │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  Step 5.3: 状态切换混合                                                      │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │ IF StateMachine.IsTransitioning()                                    │    │
│  │     BlendAlpha = StateMachine.GetBlendAlpha()                        │    │
│  │     Position = Lerp(PreviousStatePosition, CurrentPosition, Alpha)  │    │
│  │     Rotation = Slerp(PreviousStateRotation, CurrentRotation, Alpha) │    │
│  │     Distance = Lerp(PreviousStateDistance, CurrentDistance, Alpha)  │    │
│  │     FOV = Lerp(PreviousStateFOV, CurrentFOV, Alpha)                  │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  Step 5.4: 计算最终相机Transform                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │ CameraPosition = FocusPoint + (Rotation. Inverse * Distance)         │    │
│  │ CameraPosition += SocketOffset (屏幕空间偏移)                        │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  输出：FSoulsCameraOutput（统一的相机参数）                                  │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

Stage 6: Collision Resolver（碰撞解析）

┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 6: COLLISION RESOLVER                                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  职责：检测并解决相机与场景的碰撞，确保相机不穿墙                            │
│                                                                              │
│  处理流程：                                                                  │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  Step 6. 1: 碰撞检测                                                          │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │                                                                       │    │
│  │     FocusPoint ●───────────────────────────────● DesiredCameraPos   │    │
│  │                 \   多射线检测区域              /                     │    │
│  │                  \  ┌─────────────────────┐   /                      │    │
│  │                   \ │ ● ● ● ● ● ← 5条射线 │  /                       │    │
│  │                    \└─────────────────────┘ /                        │    │
│  │                     \                      /                         │    │
│  │                      ●────────────────────●                          │    │
│  │                                                                       │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  Step 6.2: 碰撞响应                                                          │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │ IF 检测到碰撞                                                        │    │
│  │     · 计算碰撞点                                                     │    │
│  │     · 相机位置 = 碰撞点 - 碰撞法线 × 安全距离                        │    │
│  │     · 记录碰撞距离                                                   │    │
│  │     · 重置恢复计时器                                                 │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  Step 6.3: 遮挡物处理                                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │ IF 焦点被遮挡                                                        │    │
│  │     FOR each 遮挡Actor                                               │    │
│  │         IF Actor支持渐隐                                             │    │
│  │             · 添加到FadedActors列表                                  │    │
│  │             · 开始渐隐动画                                           │    │
│  │         ELSE                                                          │    │
│  │             · 强制拉近相机                                           │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
