四、响应策略详细说明

┌─────────────────────────────────────────────────────────────────────────────┐
│                       RESPONSE STRATEGIES (响应策略)                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  RS01: PullIn (拉近)                                                         │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · 方法：将相机拉近到碰撞点前方                                              │
│  · 公式：SafePos = HitPoint - HitNormal × MinDistanceFromSurface            │
│  · 最常用的响应方式                                                          │
│                                                                              │
│         FocusPoint ●━━━━━━━━━●━━━━╳━━━━━● DesiredPos                        │
│                            ▲   碰撞点                                        │
│                         SafePos                                              │
│                                                                              │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  RS02: Slide (滑动)                                                          │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · 方法：沿碰撞表面滑动，尽量保持期望距离                                    │
│  · 公式：SlideDir = DesiredDir - (DesiredDir · HitNormal) × HitNormal       │
│  · 适用：墙壁等垂直表面                                                      │
│                                                                              │
│                    墙壁                                                      │
│                      │                                                       │
│         FocusPoint ●─┼──→ SlidePos                                          │
│                      │  ↗                                                    │
│                      │ DesiredPos                                            │
│                                                                              │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  RS03: Orbit (轨道调整)                                                      │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · 方法：保持距离，绕FocusPoint旋转到无碰撞位置                              │
│  · 适用：锁定状态、需要保持固定距离                                          │
│  · 搜索方向：优先水平方向，然后垂直方向                                      │
│                                                                              │
│                    墙壁                                                      │
│                      │    ● SafePos                                          │
│                      │   ╱                                                   │
│         FocusPoint ●─┼──╳ DesiredPos                                        │
│                      │                                                       │
│                                                                              │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  RS04: FOVCompensate (FOV补偿)                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · 方法：拉近时增加FOV，保持视野范围                                         │
│  · 公式：CompensatedFOV = BaseFOV × (DesiredDist / ActualDist)              │
│  · 上限：MaxFOV（防止过度畸变）                                              │
│  · 适用：配合RS01使用                                                        │
│                                                                              │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  RS05: InstantSnap (即时跳转)                                                │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · 方法：直接跳转到安全位置，无平滑过渡                                      │
│  · 适用：紧急情况、相机即将完全穿透                                          │
│  · 阈值：当碰撞距离 < MinSafeDistance时触发                                  │
│                                                                              │

五、遮挡处理详细说明

┌─────────────────────────────────────────────────────────────────────────────┐
│                       OCCLUSION HANDLING (遮挡处理)                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  遮挡 vs 碰撞的区别：                                                        │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · 碰撞：相机会穿透的静态几何体（墙、地面）                                  │
│  · 遮挡：挡在相机和玩家之间的物体（柱子、树木、NPC）                         │
│                                                                              │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  OC01: FadeOut (渐隐)                                                        │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · 方法：将遮挡物的材质渐变为透明                                            │
│  · 条件：遮挡物需要支持透明度变化                                            │
│  · 参数：FadeSpeed = 5. 0, MinAlpha = 0.3                                    │
│  · 优点：相机位置不变，视觉干扰小                                            │
│  · 缺点：需要遮挡物支持、可能有性能开销                                      │
│                                                                              │
│         Camera ●━━━━━━[遮挡物变透明]━━━━━━● Player                           │
│                                                                              │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  OC02: Hide (隐藏)                                                           │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · 方法：直接隐藏遮挡物                                                      │
│  · 条件：遮挡物标记为可隐藏                                                  │
│  · 优点：简单直接                                                            │
│  · 缺点：视觉上突然消失                                                      │
│  · 适用：小型可破坏物、草丛等                                                │
│                                                                              │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  OC03: PullInFurther (进一步拉近)                                            │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · 方法：当无法透明化时，将相机拉到遮挡物前方                                │
│  · 适用：不支持透明化的遮挡物                                                │
│  · 注意：可能导致相机过近                                                    │
│                                                                              │
│         Camera ●━━[遮挡物]━━━━━━━━━━━━━━━● Player                           │
│              ↑                                                               │
│           拉到这里                                                           │
│                                                                              │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  OC04: DitherFade (抖动渐隐)                                                 │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · 方法：使用抖动透明而非真透明                                              │
│  · 优点：性能更好（不需要透明排序）                                          │
│  · 缺点：视觉效果略差                                                        │
│  · 适用：大量遮挡物、性能敏感场景                                            │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

六、恢复策略详细说明

┌─────────────────────────────────────────────────────────────────────────────┐
│                       RECOVERY STRATEGIES (恢复策略)                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  恢复问题：碰撞解除后，如何平滑恢复到理想位置                                │
│                                                                              │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  RC01: DelayedRecovery (延迟恢复)                                            │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · 方法：碰撞解除后等待一段时间再开始恢复                                    │
│  · 参数：RecoveryDelay = 0.2s                                               │
│  · 原因：防止相机在碰撞边缘反复抖动                                          │
│                                                                              │
│      碰撞中    碰撞解除      开始恢复                                        │
│         │         │            │                                             │
│         ▼         ▼            ▼                                             │
│  ──────────────────────────────────────▶ Time                               │
│         [拉近]  [等待0.2s]  [平滑恢复]                                       │
│                                                                              │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  RC02: SmoothRecovery (平滑恢复)                                             │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · 方法：使用指数衰减插值恢复到理想距离                                      │
│  · 公式：Alpha = 1 - Exp(-RecoverySpeed × DeltaTime)                        │
│  · 参数：RecoverySpeed = 3.0                                                │
│  · 恢复速度比进入碰撞时慢，感觉更自然                                        │
│                                                                              │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  RC03: StepRecovery (分段恢复)                                               │
│  ─────────────────────────────────────────────────────────────────────────  │
│  · 方法：分多个阶段恢复，每阶段速度不同                                      │
│  · 阶段1 (0-30%): 慢速恢复                                                  │
│  · 阶段2 (30-80%): 正常速度                                                 │
│  · 阶段3 (80-100%): 慢速趋近                                                │
