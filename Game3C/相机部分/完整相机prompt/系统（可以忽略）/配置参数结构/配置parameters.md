二、参数分组架构


┌─────────────────────────────────────────────────────────────────────────────┐
│                           参数分组架构                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  DataTable设计思路：                                                         │
│  ─────────────────────────────────────────────────────────────────────────  │
│                                                                              │
│  方案A: 单一大表（所有参数在一个结构体）                                     │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │  优点:  简单直接                                                      │    │
│  │  缺点:  结构体过大，难以维护                                          │    │
│  │  适用:  小型项目                                                      │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  方案B: 分组嵌套（推荐）                                                     │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │  主结构体包含多个子结构体                                            │    │
│  │  · FCameraStateConfig (主)                                           │    │
│  │    ├─ FStateBaseParams (状态基础)                                    │    │
│  │    ├─ FModuleParams (模块参数)                                       │    │
│  │    ├─ FModifierParams (修改器参数)                                   │    │
│  │    └─ FCollisionParams (碰撞参数)                                    │    │
│  │  优点: 结构清晰，易于维护和扩展                                      │    │
│  │  适用: 中大型项目（推荐）                                            │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  方案C:  多表关联                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐    │
│  │  多个独立DataTable，通过ID关联                                       │    │
│  │  · StateBaseTable (状态基础表)                                       │    │
│  │  · ModuleParamsTable (模块参数表)                                    │    │
│  │  · ModifierParamsTable (修改器参数表)                                │    │
│  │  · CollisionParamsTable (碰撞参数表)                                 │    │
│  │  优点: 最灵活，可独立修改                                            │    │
│  │  缺点: 需要额外的关联逻辑                                            │    │
│  │  适用: 大型项目，需要独立迭代各部分                                  │    │
│  └─────────────────────────────────────────────────────────────────────┘    │
│                                                                              │
│  本清单采用方案B的结构组织参数                                               │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
