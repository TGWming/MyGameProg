```cpp
/**
 * ShouldActivate - Determine if boss look-at rotation should be active
 * 
 * R05 activates when:
 * - Module is enabled
 * - Player has a locked-on target
 * - Target IS a boss (this is the key condition)
 * 
 * R05 has the highest priority (160) among lock-on rotation modules. 
 * This ensures boss encounters get special camera treatment.
 * 
 * Differences from R03:
 * - R03 (150): Standard enemies, standard height offset
 * - R05 (160): Boss enemies, dynamic height adjustment based on size
 * 
 * Use Cases:
 * - Large boss fights (dragons, giants, etc.)
 * - Bosses with unusual proportions
 * - Epic encounters requiring cinematic framing
 */
bool UCameraModule_R05_LookAt_Boss::ShouldActivate(
	const FStageExecutionContext& Context, 
	const FCameraStateConfig& StateConfig) const
{
	// Check if module is enabled
	if (! bIsEnabled)
	{
		return false;
	}
	
	// Check if player has a locked target
	if (!Context.InputContext.bHasTarget)
	{
		return false;
	}
	
	// ONLY activate for boss targets
	// This is the key difference from R03
	if (! Context.InputContext.bTargetIsBoss)
	{
		return false;
	}
	
	// All conditions met - activate boss look-at
	return true;
}
```

```cpp
/**
 * Compute - Calculate rotation to face boss with size-based adjustments
 * 
 * Boss fights require special camera handling due to:
 * - Larger enemy models (need to look higher)
 * - Variable sizes (dragon vs humanoid boss)
 * - Dynamic framing needs (keep boss head/weak points visible)
 * 
 * Steps:
 * 1. Validate boss target exists
 * 2. Get camera focus point and boss location
 * 3. Calculate dynamic height offset based on boss size
 * 4. Apply size-to-distance ratio adjustment
 * 5. Calculate look-at rotation
 * 6. Apply pitch adjustment for very large bosses
 * 7. Clamp pitch and output
 * 
 * Key Features:
 * - Dynamic height offset (40% of boss size, 50cm-300cm range)
 * - Pitch adjustment for close large bosses
 * - Ensures boss weak points (head) are visible
 * - Adapts to different boss sizes automatically
 * 
 * @param Context The current pipeline execution context
 * @param StateConfig The current camera state configuration
 * @param OutOutput Output structure to fill with computed values
 * @return True if computation succeeded
 */
bool UCameraModule_R05_LookAt_Boss::Compute(
	const FStageExecutionContext& Context, 
	const FCameraStateConfig& StateConfig, 
	FModuleOutput& OutOutput)
{
	// Create base output with module identity
	OutOutput = CreateBaseOutput();
	
	// Step 1: Validate that we have a locked boss target
	// Safety check - ShouldActivate should prevent this, but double-check
	if (!Context. InputContext.bHasTarget)
	{
		// No target - cannot compute boss look-at rotation
		OutOutput.bHasRotationOutput = false;
		return false;
	}
	
	// Step 2: Get camera focus point
	FVector CameraFocusPoint = Context.Output.  FocusPoint;
	
	// Fallback if FocusPoint is invalid
	if (CameraFocusPoint.  IsNearlyZero())
	{
		CameraFocusPoint = Context. InputContext.CharacterLocation;
		CameraFocusPoint. Z += 100.0f;
	}
	
	// Step 3: Get boss location and size
	FVector TargetLocation = Context.InputContext.  TargetLocation;
	float TargetSize = Context.InputContext.TargetSize;
	
	// If TargetSize not available, use default large boss size
	if (TargetSize <= 0.0f)
	{
		TargetSize = 200.0f;  // Default:  2 meters (large enemy)
	}
	
	// Step 4: Calculate dynamic height offset based on boss size
	// Boss look-at aims at center mass or upper body
	// Larger bosses = look higher (to see head/weak points)
	// 
	// Formula: 40% of boss size
	// - Small boss (150cm): 60cm offset (chest level)
	// - Medium boss (300cm): 120cm offset (head level)
	// - Large boss (500cm): 200cm offset (upper body)
	// - Huge boss (1000cm): 300cm offset (capped, prevent looking too high)
	float BossHeightOffset = TargetSize * 0.4f;
	
	// Clamp to reasonable range
	// Min 50cm:  ensure we don't look at feet even for small bosses
	// Max 300cm: prevent looking too high for giant bosses
	BossHeightOffset = FMath::Clamp(BossHeightOffset, 50.0f, 300.0f);
	
	// Apply height offset to boss location
	FVector BossLookPoint = TargetLocation;
	BossLookPoint.Z += BossHeightOffset;
	
	// Step 5: Calculate base look direction
	FVector LookDirection = (BossLookPoint - CameraFocusPoint).GetSafeNormal();
	
	// Handle edge case: zero direction
	if (LookDirection. IsNearlyZero())
	{
		LookDirection = FVector::  ForwardVector;
	}
	
	// Convert to rotation
	FRotator LookAtRotation = LookDirection.Rotation();
	
	// Step 6: Apply dynamic pitch adjustment for very large bosses
	// When boss is large relative to distance, tilt up slightly
	// This helps keep more of the boss visible in frame
	
	float DistanceToTarget = Context. InputContext.  TargetDistance;
	
	// If DistanceToTarget not available, calculate from positions
	if (DistanceToTarget <= 0.0f)
	{
		DistanceToTarget = (TargetLocation - Context.InputContext.CharacterLocation).Size();
	}
	
	// Calculate size-to-distance ratio
	// High ratio = boss is large relative to how far away it is
	// Example: 500cm boss at 1000cm distance → ratio = 0.5
	float SizeToDistanceRatio = TargetSize / FMath::Max(DistanceToTarget, 100.0f);
	
	// If ratio > 0.5 (boss is more than half the distance in size)
	// Apply additional pitch adjustment to tilt camera up
	if (SizeToDistanceRatio > 0.5f)
	{
		// Boss is large relative to distance, tilt up slightly
		// This prevents boss from filling entire frame awkwardly
		// 
		// Formula: (Ratio - 0.5) * 20 degrees
		// - Ratio 0.5: 0 degrees (no adjustment)
		// - Ratio 0.75: 5 degrees up
		// - Ratio 1.0: 10 degrees up
		// - Ratio 1.5: 15 degrees up (capped)
		float PitchAdjust = (SizeToDistanceRatio - 0.5f) * 20.0f;
		
		// Clamp to max 15 degrees
		PitchAdjust = FMath:: Clamp(PitchAdjust, 0.0f, 15.0f);
		
		// Apply adjustment (subtract to tilt up, negative pitch = up)
		LookAtRotation.Pitch -= PitchAdjust;
	}
	
	// Step 7: Clamp pitch to configured limits
	float MinPitch = StateConfig.StateBase. Rotation.  MinPitch;
	float MaxPitch = StateConfig.StateBase.Rotation.MaxPitch;
	
	// Validate pitch limits are reasonable
	if (MinPitch >= MaxPitch)
	{
		MinPitch = -80.0f;
		MaxPitch = 80.0f;
	}
	
	LookAtRotation.Pitch = FMath::Clamp(LookAtRotation.Pitch, MinPitch, MaxPitch);
	
	// Step 8: Force roll to zero
	LookAtRotation.Roll = 0.0f;
	
	// Step 9: Fill output structure
	OutOutput.  RotationOutput = LookAtRotation;
	OutOutput. bHasRotationOutput = true;
	
	return true;
}
```

**说明：**
- 动态高度偏移（40%，50-300cm范围）
- 尺寸-距离比例调整（防止Boss填满屏幕）
- Pitch微调（大型Boss额外上仰）
- 与R03类似的基础结构

【技术说明】

**R05的核心机制：**

### 1. 动态高度偏移

**为什么使用40%而非30%（R03）？**

```
R03 (普通敌人):
TargetHeightOffset = TargetSize * 0.3f

- 适用于人形敌人（100-150cm）
- 偏移30-45cm（胸部/头部）
- 标准视角

R05 (Boss):
BossHeightOffset = TargetSize * 0.4f

- 适用于大型Boss（200-1000cm）
- 偏移80-300cm（更高）
- 看向Boss上半身/头部/弱点
```

**不同Boss尺寸的效果：**

| Boss类型 | TargetSize | 40%偏移 | 限制后 | 注视点 |
|---------|-----------|---------|--------|--------|
| 小型Boss | 150cm | 60cm | 60cm | 上胸 |
| 中型Boss | 300cm | 120cm | 120cm | 头部 |
| 大型Boss | 500cm | 200cm | 200cm | 头顶 |
| 巨型Boss | 1000cm | 400cm | 300cm ✅ | 上身（限制） |

**Clamp的必要性：**
```cpp
BossHeightOffset = FMath::Clamp(BossHeightOffset, 50.0f, 300.0f);

不限制：
- 巨型Boss（1000cm）→ 偏移400cm
- 相机看向Boss头顶很高的位置
- Boss主体不在视野内（awkward）

限制到300cm：
- 即使巨型Boss，偏移也只有300cm
- 保持Boss主体可见
- 合理的上限
```

### 2. 尺寸-距离比例调整

**为什么需要SizeToDistanceRatio？**

```
场景1：中型Boss，远距离
TargetSize = 300cm
Distance = 1500cm
Ratio = 300 / 1500 = 0.2

→ 比例低（<0.5）
→ Boss在远处，看起来小
→ 不需要额外调整
→ PitchAdjust = 0

场景2：大型Boss，近距离
TargetSize = 800cm
Distance = 1000cm
Ratio = 800 / 1000 = 0.8

→ 比例高（>0.5）
→ Boss填满大部分视野
→ 需要稍微上仰
→ PitchAdjust = (0.8 - 0.5) * 20 = 6°
→ 相机上仰6°，看到Boss全貌

场景3：巨型Boss，很近
TargetSize = 1000cm
Distance = 800cm
Ratio = 1000 / 800 = 1.25

→ 比例很高（>1.0）
→ Boss非常巨大
→ PitchAdjust = (1.25 - 0.5) * 20 = 15°（限制到15°）
→ 相机上仰15°（最大），尽量展现Boss
```

**数学公式：**
```
Ratio = TargetSize / Distance

if Ratio > 0.5:
    PitchAdjust = (Ratio - 0.5) * 20°
    Clamp(PitchAdjust, 0°, 15°)
    
    FinalPitch = BasePitch - PitchAdjust

解释：
- Ratio越高，Boss相对越大
- PitchAdjust越大，相机越上仰
- 最多上仰15°（防止过度）
```

**为什么阈值是0.5？**

```
Ratio = 0.5 意味着：
Boss尺寸 = 距离的一半

示例：
Boss高5米，距离10米 → Ratio=0.5

视觉效果：
- Boss占据视野的显著部分
- 但不是压倒性的
- 这是一个平衡点

Ratio < 0.5:
- Boss相对较小，标准视角足够

Ratio > 0.5:
- Boss相对较大，需要调整
```
