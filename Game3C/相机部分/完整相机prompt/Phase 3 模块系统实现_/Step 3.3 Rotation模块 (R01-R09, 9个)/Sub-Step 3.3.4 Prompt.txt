/**
 * ShouldActivate - Determine if hard lock-on rotation should be active
 * 
 * R03 activates when:
 * - Module is enabled
 * - Player has a locked-on target
 * - Target is NOT a boss (boss uses R05 with different parameters)
 * - Current state requires target tracking (bRequiresTarget flag)
 * 
 * R03 is the core lock-on module for standard combat. 
 * It has higher priority (150) than input modules (100), so it
 * overrides player input when a target is locked.
 * 
 * Use Cases:
 * - Dark Souls style lock-on combat
 * - Precise target tracking during fights
 * - Keeping enemies centered in view
 */
bool UCameraModule_R03_LookAt_Target::ShouldActivate(
	const FStageExecutionContext& Context, 
	const FCameraStateConfig& StateConfig) const
{
	// Check if module is enabled
	if (!bIsEnabled)
	{
		return false;
	}
	
	// Check if player has a locked target
	// This is the primary activation condition
	if (!Context. InputContext.bHasTarget)
	{
		return false;
	}
	
	// Don't activate for boss targets (R05 handles those)
	// Boss targets need different camera behavior
	if (Context.InputContext.bTargetIsBoss)
	{
		return false;
	}
	
	// Check if current state requires target tracking
	// Some states may have targets but don't want lock-on camera
	if (! StateConfig.StateBase.Flags. bRequiresTarget)
	{
		return false;
	}
	
	// All conditions met - activate hard lock-on
	return true;
}
```

**说明：**
- 四重条件检查：启用 + 有目标 + 非Boss + 需要跟踪
- 与R01/R02互斥（bHasTarget）
- 与R05互斥（bTargetIsBoss）
- 优先级150高于输入模块100

/**
 * Compute - Calculate rotation to face locked target
 * 
 * This module calculates the rotation needed to make the camera face
 * the locked target directly.  It's a "hard" lock, meaning the camera
 * always points exactly at the target, ignoring player input.
 * 
 * Steps:
 * 1. Validate target exists
 * 2. Get camera focus point (from Position modules)
 * 3. Get target location (from InputContext)
 * 4. Apply height offset to target (look at upper body, not feet)
 * 5. Calculate look direction vector
 * 6. Convert to rotation
 * 7. Clamp pitch to limits
 * 8. Output the calculated rotation
 * 
 * Key Design:
 * - Camera focus point is calculated by Position modules (P01-P08)
 * - We calculate rotation FROM focus point TO target
 * - This ensures correct look-at even with complex position logic
 * 
 * Height Offset:
 * - Without offset: camera looks at target's feet (awkward)
 * - With offset: camera looks at target's center/head (natural)
 * - Offset scales with target size (larger enemies = higher offset)
 * 
 * @param Context The current pipeline execution context
 * @param StateConfig The current camera state configuration
 * @param OutOutput Output structure to fill with computed values
 * @return True if computation succeeded
 */
bool UCameraModule_R03_LookAt_Target::Compute(
	const FStageExecutionContext& Context, 
	const FCameraStateConfig& StateConfig, 
	FModuleOutput& OutOutput)
{
	// Create base output with module identity
	OutOutput = CreateBaseOutput();
	
	// Step 1: Validate that we have a locked target
	// Safety check - ShouldActivate should prevent this, but double-check
	if (!Context.InputContext.bHasTarget)
	{
		// No target - cannot compute look-at rotation
		OutOutput.bHasRotationOutput = false;
		return false;
	}
	
	// Step 2: Get camera focus point
	// This is calculated by Position modules in Stage 3
	// It represents where the camera is positioned/looking from
	FVector CameraFocusPoint = Context.Output.FocusPoint;
	
	// If FocusPoint is zero/invalid, use fallback
	if (CameraFocusPoint.IsNearlyZero())
	{
		// Fallback:  use character location as focus point
		CameraFocusPoint = Context.InputContext.CharacterLocation;
		
		// Add standard height offset for typical camera position
		CameraFocusPoint. Z += 100.0f;  // 1 meter above character
	}
	
	// Step 3: Get target location from InputContext
	FVector TargetLocation = Context. InputContext.TargetLocation;
	
	// Step 4: Apply height offset to target
	// This makes the camera look at the target's center/upper body
	// rather than their feet, which looks more natural
	
	// Get target size (typically capsule radius or bounding sphere radius)
	float TargetSize = Context.InputContext.TargetSize;
	
	// If TargetSize not available, use default humanoid height
	if (TargetSize <= 0.0f)
	{
		TargetSize = 100.0f;  // Default: 1 meter (typical character)
	}
	
	// Height offset: 30% of target size
	// This typically aims at chest/head level for humanoids
	// Example: TargetSize=100cm → Offset=30cm (chest level)
	float TargetHeightOffset = TargetSize * 0.3f;
	
	// Clamp offset to reasonable range (don't look too high)
	TargetHeightOffset = FMath::Clamp(TargetHeightOffset, 20.0f, 100.0f);
	
	// Apply height offset to target location
	TargetLocation.Z += TargetHeightOffset;
	
	// Step 5: Calculate look direction vector
	// Direction FROM camera focus point TO adjusted target location
	FVector LookDirection = TargetLocation - CameraFocusPoint;
	
	// Normalize to unit vector (GetSafeNormal handles zero-length safely)
	LookDirection = LookDirection.GetSafeNormal();
	
	// Handle edge case: if direction is zero (camera at target position)
	if (LookDirection.IsNearlyZero())
	{
		// Fallback: look forward
		LookDirection = FVector::ForwardVector;
	}
	
	// Step 6: Convert direction vector to rotation
	// Rotation() method converts a direction vector to FRotator
	FRotator LookAtRotation = LookDirection.Rotation();
	
	// Step 7: Clamp pitch to configured limits
	// Same as input modules, prevent extreme up/down angles
	float MinPitch = StateConfig.StateBase.Rotation.MinPitch;
	float MaxPitch = StateConfig.StateBase.Rotation.MaxPitch;
	
	// Validate pitch limits are reasonable
	if (MinPitch >= MaxPitch)
	{
		// Invalid config, use default limits
		MinPitch = -80.0f;
		MaxPitch = 80.0f;
	}
	
	LookAtRotation. Pitch = FMath:: Clamp(LookAtRotation.Pitch, MinPitch, MaxPitch);
	
	// Step 8: Force roll to zero
	// Third-person cameras should never roll
	LookAtRotation. Roll = 0.0f;
	
	// Step 9: Fill output structure
	OutOutput.RotationOutput = LookAtRotation;
	OutOutput.bHasRotationOutput = true;
	
	return true;
}
```

**说明：**
- 从FocusPoint计算朝向（依赖Position模块）
- 目标高度偏移（避免看向脚部）
- 使用GetSafeNormal（防止除零）
- Pitch限制（与输入模块一致）
- Roll强制为零

	OutOutput. bHasRotationOutput = true;

【技术说明】

**R03的核心机制：**

### 1. Look-At计算原理

**数学基础：**
```
给定：
- 相机焦点位置：CameraFocusPoint (Fc)
- 目标位置：TargetLocation (Tl)

计算：
- 朝向向量：LookDirection = Tl - Fc
- 归一化：LookDirection = LookDirection. Normalize()
- 转换为旋转：LookAtRotation = LookDirection. Rotation()

示例：
Fc = (0, 0, 200)       // 相机在角色上方2米
Tl = (100, 0, 150)     // 目标在前方1米，略低

LookDirection = (100, 0, -50) - (0, 0, 0) = (100, 0, -50)
Normalize → (0.894, 0, -0.447)
Rotation() → Yaw≈0°, Pitch≈-26.5°

结果：相机向前下方26.5°看（朝向目标）
```

### 2. 高度偏移的重要性

**为什么需要TargetHeightOffset？**

```
场景：玩家锁定人形敌人

不加高度偏移：
TargetLocation = 敌人位置 (例如 100, 200, 0)
→ 这是敌人脚底位置
→ 相机看向敌人脚部（awkward！）

加高度偏移（30%）：
TargetSize = 100cm（人形敌人高度）
TargetHeightOffset = 100 * 0.3 = 30cm
TargetLocation.Z += 30
→ 相机看向敌人腰部/胸部（natural！）
```

**不同偏移比例的效果：**

| 偏移比例 | 效果 | 适用场景 |
|---------|------|---------|
| 0% | 看脚部 | 不推荐 |
| 20% | 看腿部 | 不自然 |
| 30% | 看胸部 | ✅ 推荐（人形） |
| 50% | 看头部 | 较高视角 |
| 100% | 看头顶 | 巨型敌人 |

### 3. 与Position模块的配合

**Pipeline流程：**
```
Stage 3 (Module Compute):

1. Position模块执行（P01-P08）
   → 计算FocusPoint
   → 存储到Context.Output.FocusPoint

2. Rotation模块执行（R01-R09）
   → R03读取Context.Output.FocusPoint
   → 从FocusPoint看向Target
   → 输出RotationOutput

关键：
- Position决定"从哪里看"（FocusPoint）
- Rotation决定"看向哪里"（RotationOutput）
- 两者分离，灵活组合
```

**示例场景：**
```
场景：P04 (Orbit Lock-On) + R03 (LookAt Target)

P04计算：
FocusPoint = 玩家和目标的中点

R03计算：
从FocusPoint看向Target
→ 相机焦点在中点
→ 相机朝向指向目标
→ 效果：保持双方可见，同时面向目标
```

### 5. GetSafeNormal的作用

**为什么使用GetSafeNormal？**

```cpp
// 问题：零长度向量
FVector Direction = TargetLocation - CameraFocusPoint;
Direction. Normalize();  // ❌ 如果长度为0，会除零崩溃

// 解决：安全归一化
FVector Direction = (TargetLocation - CameraFocusPoint).GetSafeNormal();
// ✅ 如果长度为0，返回(0,0,0)而不崩溃

// 边缘情况：相机和目标在同一位置
CameraFocusPoint = (100, 200, 150)
TargetLocation = (100, 200, 150)
Direction = (0, 0, 0)
GetSafeNormal() → (0, 0, 0)

// 处理边缘情况
if (Direction.IsNearlyZero())
{
	Direction = FVector::ForwardVector;  // Fallback
}
```

### 6. FVector. Rotation()方法

**自动转换方向到旋转：**
