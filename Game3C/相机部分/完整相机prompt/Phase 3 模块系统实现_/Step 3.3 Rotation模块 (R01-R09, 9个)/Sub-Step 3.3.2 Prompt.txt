/**

 * 
 * R01 activates when:
 * - Module is enabled
 * - No target is locked (free camera, not lock-on)
 * - Input is not explicitly ignored (StateConfig flag)
 *

	const FStageExecutionContext& Context, 
	const FCameraStateConfig& StateConfig) const
{

	if (!bIsEnabled)
	{
		return false;

	if (Context.  InputContext. bHasTarget)

**完整实现：**
```cpp
/**
 * Compute - Calculate camera rotation from player input
 * 
 * This module processes raw player input and converts it to camera rotation.
 * 
 * Steps:
 * 1. Get camera input from InputContext (X=Yaw, Y=Pitch)
 * 2. Apply sensitivity multiplier
 * 3. Accumulate rotation deltas
 * 4. Clamp pitch to configured limits
 * 5. Normalize yaw to 0-360 range
 * 6. Output the final rotation
 * 
 * Input Conventions:
 * - CameraInput.X:   Horizontal input (Yaw) → Left/Right
 * - CameraInput.Y:  Vertical input (Pitch) → Up/Down
 * - Pitch is inverted for standard third-person controls
 * 
 * @param Context The current pipeline execution context
 * @param StateConfig The current camera state configuration
 * @param OutOutput Output structure to fill with computed values
 * @return True if computation succeeded
 */
bool UCameraModule_R01_PlayerInput_Free::  Compute(
	const FStageExecutionContext& Context, 
	const FCameraStateConfig& StateConfig, 
	FModuleOutput& OutOutput)
{
	// Create base output with module identity
	OutOutput = CreateBaseOutput();
	
	// Step 1: Get camera input from InputContext
	// CameraInput is a 2D vector:  X=Yaw (horizontal), Y=Pitch (vertical)
	FVector2D CameraInput = Context.InputContext.CameraInput;
	
	// Step 2: Get sensitivity from state config
	// InputSensitivity is a global multiplier, BaseSensitivity is per-module
	float InputSensitivity = StateConfig. Module.  Rotation.InputSensitivity;
	
	// If InputSensitivity not configured, use default
	if (InputSensitivity <= 0.0f)
	{
		InputSensitivity = 1.0f;
	}
	
	float TotalSensitivity = InputSensitivity * BaseSensitivity;
	
	// Step 3: Calculate rotation deltas from input
	// Yaw (horizontal rotation, left-right)
	float DeltaYaw = CameraInput.X * TotalSensitivity;
	
	// Pitch (vertical rotation, up-down)
	// Multiply by -1.0 to invert (standard third-person convention)
	// Positive input (up) → negative pitch (look up)
	float DeltaPitch = CameraInput.Y * TotalSensitivity * -1.0f;
	
	// Step 4: Accumulate deltas to current rotation
	// This maintains rotation state across frames
	CurrentRotation. Yaw += DeltaYaw;
	CurrentRotation. Pitch += DeltaPitch;
	
	// Step 5: Clamp pitch to configured limits
	// Prevents camera from flipping over or looking straight down
	// Typical values: MinPitch=-80°, MaxPitch=80°
	float MinPitch = StateConfig.  StateBase.Rotation.MinPitch;
	float MaxPitch = StateConfig. StateBase. Rotation.MaxPitch;
	
	// Validate pitch limits are reasonable
	if (MinPitch >= MaxPitch)
	{
		// Invalid config, use default limits
		MinPitch = -80.0f;
		MaxPitch = 80.0f;
	}
	
	CurrentRotation.Pitch = FMath::Clamp(CurrentRotation.Pitch, MinPitch, MaxPitch);
	
	// Step 6: Normalize yaw to 0-360 range
	// Prevents yaw from growing infinitely large
	// fmod handles both positive and negative values correctly
	CurrentRotation.Yaw = FMath::  Fmod(CurrentRotation. Yaw, 360.0f);
	if (CurrentRotation.Yaw < 0.0f)
	{
		CurrentRotation.  Yaw += 360.0f;
	}
	
	// Step 7: Force roll to zero
	// Third-person cameras should never roll
	CurrentRotation. Roll = 0.0f;
	
	// Step 8: Fill output structure
	OutOutput. RotationOutput = CurrentRotation;
	OutOutput.bHasRotationOutput = true;
	
	return true;
}
```

**说明：**
- 详细的步骤注释
- 输入约定说明（Pitch反转）
- Pitch限制和Yaw归一化
- Roll强制为零（第三人称标准）
- 完整的默认值处理

【技术说明】

**R01的核心机制：**

### 1. 输入到旋转的转换

**输入约定：**
```
CameraInput.X:  水平输入（Yaw）
- 正值 = 向右旋转
- 负值 = 向左旋转

CameraInput.Y:  垂直输入（Pitch）
- 正值 = 向上输入
- 负值 = 向下输入

但是！Pitch需要反转：
- 向上输入（正值）→ 向上看（负Pitch）
- 向下输入（负值）→ 向下看（正Pitch）

这是第三人称标准控制约定
```

### 2. 旋转累积

**为什么需要CurrentRotation？**
```
问题：如果每帧只处理DeltaRotation
- 无法保持相机朝向
- 每帧都会重置
- 失去旋转的连续性

解决：维护CurrentRotation状态
- 每帧累积DeltaRotation
- CurrentRotation += Delta
- 保持相机朝向的连续性
```

### 3. Pitch限制

**为什么需要限制Pitch？**
```
问题：不限制Pitch会导致：
- Pitch=-90°:  相机直视正下方（万向锁）
- Pitch=90°:   相机直视正上方（万向锁）
- Pitch>90° 或 <-90°:  相机翻转（上下颠倒）

解决：Clamp到合理范围
- 典型值：-80° 到 +80°
- 避免万向锁和翻转
- 保持相机稳定

实现：
CurrentRotation.Pitch = Clamp(Pitch, MinPitch, MaxPitch);
```

### 4. Yaw归一化

**为什么需要归一化Yaw？**
```
问题：不归一化Yaw会导致：
- 持续旋转：Yaw可以增长到无限大
- 360° = 720° = 1080° = ... 
- 浮点精度问题

解决：归一化到0-360°
Yaw = Fmod(Yaw, 360);
if (Yaw < 0) Yaw += 360;

效果：
- 365° → 5°
- -10° → 350°
- 保持语义等价，避免精度损失
```

### 5. Roll强制为零

**为什么Roll必须为零？**
```
第三人称相机不应该Roll（倾斜）
- Roll=0°: 水平正常
- Roll≠0°: 倾斜（让人晕眩）

特例：
- 某些动作游戏可能在特殊时刻使用Roll
- 但标准第三人称相机Roll=0

实现：
CurrentRotation.Roll = 0.0f;  // 强制
```

### 6. 灵敏度系统

**两级灵敏度：**
```
BaseSensitivity (模块级):
- 每个旋转模块可以有自己的基础灵敏度
- R01可能是1.0（标准）
- R02可能是0.8（更柔和）

InputSensitivity (配置级):
- 来自StateConfig
- 玩家可调整
- 应用到所有旋转模块

TotalSensitivity = BaseSensitivity * InputSensitivity
```
