```cpp
/**
 * ShouldActivate - Determine if aim FOV reduction should be active
 * 
 * F03 activates when: 
 * - Module is enabled
 * - Player is in aiming/focus mode (bIsAiming)
 * - OR player has a locked target in combat
 * 
 * In Souls-like games, locking onto an enemy often triggers a slight
 * FOV reduction to create a more focused, intense combat feel. 
 * 
 * Priority 115 (higher than speed/combat), uses Additive policy. 
 * Negative offset to reduce FOV. 
 * 
 * Use Cases:
 * - Lock-on combat (common in Souls-like games)
 * - Bow/crossbow aiming
 * - Spell casting focus
 * - Precision attacks
 */
bool UCameraModule_F03_Aim_FOV:: ShouldActivate(
	const FStageExecutionContext& Context, 
	const FCameraStateConfig& StateConfig) const
{
	// Check if module is enabled
	if (!bIsEnabled)
	{
		return false;
	}
	
	// Check if player is explicitly aiming
	// This flag is typically set when using ranged weapons or focus mode
	bool bIsAiming = Context.InputContext. bIsAiming;
	
	// In Souls-like games, locking onto a target also counts as "aiming"
	// The camera focuses on the target, so FOV should narrow slightly
	bool bHasLockedTarget = Context. InputContext.bHasTarget;
	
	// Activate if either aiming or has locked target
	if (! bIsAiming && !bHasLockedTarget)
	{
		return false;
	}
	
	// All conditions met - activate aim FOV
	return true;
}
```

**C. Compute() 实现：**

```cpp
/**
 * Compute - Calculate FOV reduction for aiming/focus mode
 * 
 * This module calculates a negative FOV offset to narrow the view
 * during aiming or lock-on.  This creates a more focused, intense
 * visual experience during combat.
 * 
 * Steps:
 * 1. Determine aim intensity (full aim vs lock-on)
 * 2. Get aim FOV reduction from config
 * 3. Calculate target blend value
 * 4. Smooth the blend transition
 * 5. Calculate final FOV offset (negative)
 * 6. Output with Additive policy
 * 
 * Aim Types and Intensity:
 * - Full aim (bIsAiming): 100% FOV reduction
 * - Lock-on only: 50-70% FOV reduction (configurable)
 * 
 * Typical FOV reductions:
 * - Light focus: -5 to -10 degrees
 * - Standard aim: -10 to -15 degrees
 * - Heavy zoom: -20 to -30 degrees
 * 
 * @param Context The current pipeline execution context
 * @param StateConfig The current camera state configuration
 * @param OutOutput Output structure to fill with computed values
 * @return True if computation succeeded
 */
bool UCameraModule_F03_Aim_FOV::Compute(
	const FStageExecutionContext& Context, 
	const FCameraStateConfig& StateConfig, 
	FModuleOutput& OutOutput)
{
	// Create base output with module identity
	OutOutput = CreateBaseOutput();
	
	// Step 1: Determine aim intensity based on context
	bool bIsAiming = Context.InputContext.bIsAiming;
	bool bHasLockedTarget = Context.InputContext.bHasTarget;
	
	float TargetBlend = 0.0f;
	
	if (bIsAiming)
	{
		// Full aiming (e.g., bow, crossbow, spell targeting)
		// Apply 100% of the FOV reduction
		TargetBlend = 1.0f;
	}
	else if (bHasLockedTarget)
	{
		// Lock-on only (Souls-like target lock)
		// Apply partial FOV reduction for subtle focus effect
		// This can be configured, default to 50%
		float LockOnIntensity = StateConfig.Module. FOV.LockOnFOVIntensity;
		if (LockOnIntensity <= 0.0f)
		{
			LockOnIntensity = 0.5f;  // Default:  50% of aim reduction
		}
		TargetBlend = LockOnIntensity;
	}
	
	// Step 2: Get aim FOV reduction from config
	float AimFOVReduction = StateConfig.Module. FOV.AimFOVReduction;
	
	// If not configured, use default
	if (AimFOVReduction <= 0.0f)
	{
		AimFOVReduction = 15.0f;  // Default: 15 degrees reduction
	}
	
	// Step 3: Smooth the blend transition
	// Fast blend for responsive aiming feel
	float BlendSpeed = 8.0f;  // Quick response
	
	// Try to get blend speed from config
	float ConfigBlendSpeed = StateConfig.Module.FOV.AimFOVBlendSpeed;
	if (ConfigBlendSpeed > 0.0f)
	{
		BlendSpeed = ConfigBlendSpeed;
	}
	
	// Interpolate current blend towards target
	CurrentAimFOVBlend = FMath::FInterpTo(
		CurrentAimFOVBlend, 
		TargetBlend, 
		Context.DeltaTime, 
		BlendSpeed
	);
	
	// Step 4: Calculate FOV reduction (negative value)
	// Negative because we want to REDUCE the FOV
	float FOVOffset = -AimFOVReduction * CurrentAimFOVBlend;
	
	// Step 5: Fill output structure
	OutOutput. FOVOutput = FOVOffset;
	OutOutput.bHasFOVOutput = true;
	
	// Full weight for aim FOV
	OutOutput.Weight = 1.0f;
	
	return true;
}
```

【技术说明】

**F03的核心机制：**

### 1. 瞄准 vs 锁定

**两种触发方式的区别：**

```
bIsAiming (明确瞄准):
- 使用弓/弩/法杖
- 明确的瞄准模式
- 100% FOV减少
- 最终FOV:  90 - 15 = 75°

bHasTarget (锁定目标):
- 类魂游戏的锁定机制
- 只是关注目标，不是精确瞄准
- 50% FOV减少（可配置）
- 最终FOV: 90 - 7.5 = 82.5°

设计理念：
- 锁定时轻微聚焦，保持战斗灵活性
- 明确瞄准时强力聚焦，强调精确
```

### 2. 负偏移的含义

**为什么是负值？**

```cpp
// FOV减少 = 视野变窄 = 放大效果
float FOVOffset = -AimFOVReduction * CurrentAimFOVBlend;

// 示例：
// AimFOVReduction = 15°
// CurrentAimFOVBlend = 1.0 (完全瞄准)
// FOVOffset = -15 × 1.0 = -15°

// Stage 5混合：
// BaseFOV + Offset = 90 + (-15) = 75°
```

### 3. 平滑过渡的重要性

**为什么需要平滑？**

```
无平滑（假设）：
T=0s: 未锁定，FOV=90°
T=0.1s: 锁定目标，FOV=75°（瞬间跳变）
→ 视觉突兀，体验差

有平滑（实际）：
T=0s: 未锁定，FOV=90°
T=0.1s: 锁定，Blend=0.4，FOV=84°
T=0.2s:  Blend=0.7，FOV=79. 5°
T=0.3s: Blend=0.9，FOV=76.5°
T=0.5s: Blend≈1.0，FOV≈75°
→ 平滑过渡，专业感

BlendSpeed=8.0表示每秒变化8个单位
约0.5秒达到目标
```

### 4. 与F02(Speed FOV)的交互

**冲刺时锁定目标：**

```
场景：玩家冲刺中锁定敌人

F02 (Speed): +12°（冲刺FOV增加）
F03 (Aim): -7.5°（锁定FOV减少，50%强度）

最终偏移: +12 + (-7.5) = +4.5°
最终FOV: 90 + 4.5 = 94.5°

效果：
- 冲刺的速度感仍在
- 但锁定目标略微收窄
- 两者平衡
```

### 5. LockOnIntensity的设计

**为什么锁定强度可配置？**

```cpp
// 不同游戏风格的需求：
float LockOnIntensity = StateConfig. Module.FOV. LockOnFOVIntensity;

// 快节奏动作游戏：
LockOnIntensity = 0.3f;  // 30%，轻微聚焦

// 标准类魂游戏：
LockOnIntensity = 0.5f;  // 50%，适中聚焦

// 慢节奏策略游戏：
LockOnIntensity = 0.7f;  // 70%，明显聚焦

// 好处：
// - 可以在配置中调整
// - 不同敌人类型可以有不同强度
// - 设计师可以微调手感
```
