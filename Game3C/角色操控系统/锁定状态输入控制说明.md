# 锁定状态输入控制实现说明

## 📋 概述

已成功实现类似黑暗之魂3的锁定状态输入控制机制。该系统在锁定目标时会禁用摄像机控制输入，但保留右摇杆的目标切换功能。

---

## ✅ 实现功能

### 1. **未锁定状态（正常模式）**
所有输入正常工作：

| 输入设备 | 控制功能 |
|---------|---------|
| **鼠标 X** | 控制视角左右旋转（Yaw） → `Turn()` |
| **鼠标 Y** | 控制视角上下旋转（Pitch） → `LookUp()` |
| **手柄右摇杆 X 轴** | 控制视角左右旋转（Yaw） → `TurnRate()` |
| **手柄右摇杆 Y 轴** | 控制视角上下旋转（Pitch） → `LookUpRate()` |

### 2. **锁定状态（战斗模式）**
摄像机控制被禁用，右摇杆用于切换目标：

| 输入设备 | 控制功能 |
|---------|---------|
| **鼠标 X** | ❌ **禁用** (`Turn()` 返回) |
| **鼠标 Y** | ❌ **禁用** (`LookUp()` 返回) |
| **手柄右摇杆 Y 轴** | ❌ **禁用** (`LookUpRate()` 返回) |
| **手柄右摇杆 X 轴左** | ✅ **切换到左侧目标** (`HandleRightStickX()`) |
| **手柄右摇杆 X 轴右** | ✅ **切换到右侧目标** (`HandleRightStickX()`) |

### 3. **解锁后（手动或自动）**
输入控制立即恢复到未锁定状态，无需重启或额外操作。

## 🔧 核心实现逻辑

### 代码位置
- **文件**: `Source\MyProject\Private\Mycharacter.cpp` 和 `Source\MyProject\Public\Mycharacter.h`
- **关键函数**:
  - `Turn(float Value)` - 处理鼠标X轴视角左右旋转
  - `TurnRate(float Rate)` - 处理手柄右摇杆X轴视角左右旋转
  - `LookUp(float Value)` - 处理鼠标Y轴视角上下旋转
  - `LookUpRate(float Rate)` - 处理手柄右摇杆Y轴视角上下旋转
  - `HandleRightStickX(float Value)` - 处理右摇杆X轴目标切换
  - `StartLockOn(AActor* Target)` - 开始锁定
  - `CancelLockOn()` - 取消锁定

### 输入过滤机制

#### 鼠标输入（直接值）

```cpp
void AMycharacter::Turn(float Value)
{
    // 锁定状态下完全禁用偏航视角控制
    if (bIsLockedOn)
    {
        return; // 直接返回，不处理输入
    }
    
    // 未锁定状态：正常处理
    AddControllerYawInput(Value);
}

void AMycharacter::LookUp(float Value)
{
    // 锁定状态下完全禁用俯仰视角控制
    if (bIsLockedOn)
    {
        return; // 直接返回，不处理输入
    }
    
    // 未锁定状态：正常处理
    AddControllerPitchInput(Value);
}
```

#### 手柄输入（速率值）

```cpp
void AMycharacter::TurnRate(float Rate)
{
    // 锁定状态下完全禁用偏航视角控制
    if (bIsLockedOn)
    {
        return; // 直接返回，不处理输入
    }
    
    // 未锁定状态：帧率独立控制
    AddControllerYawInput(Rate * GetWorld()->GetDeltaSeconds());
}

void AMycharacter::LookUpRate(float Rate)
{
    // 锁定状态下完全禁用俯仰视角控制
    if (bIsLockedOn)
    {
        return; // 直接返回，不处理输入
    }
    
    // 未锁定状态：帧率独立控制
    AddControllerPitchInput(Rate * GetWorld()->GetDeltaSeconds());
}
```

### 右摇杆双功能

```cpp
void AMycharacter::HandleRightStickX(float Value)
{
    // 未锁定：不做特殊处理（通过TurnRate处理）
    if (!bIsLockedOn)
    {
        LastRightStickX = Value;
        return;
    }

    // 锁定：用于目标切换
    // 检测边缘触发（避免持续触发）
    if (向左推动)
    {
        SwitchLockOnTargetLeft();
    }
    else if (向右推动)
    {
        SwitchLockOnTargetRight();
    }
}
```

  └─ Gamepad Right Thumbstick X-Axis (Scale: 1.0) 【需要手动添加】

## 🔍 问题解决历程

### 问题：锁定状态下摄像机仍然可以旋转

**原因**：
- 在 `DefaultInput.ini` 中，手柄右摇杆同时绑定到了两组轴：
  - `Turn` 和 `LookUp` （鼠标使用）
  - `TurnRate` 和 `LookUpRate` （手柄使用）
- 原始代码只拦截了 `Turn` 和 `LookUp`，但没有拦截 `TurnRate` 和 `LookUpRate`

**解决方案**：
1. 在头文件中添加 `TurnRate()` 和 `LookUpRate()` 函数声明
2. 在 cpp 文件中实现这两个函数，添加相同的锁定检查逻辑
3. 在 `SetupPlayerInputComponent()` 中绑定这两个函数
4. 现在所有四个输入通道都被正确拦截
