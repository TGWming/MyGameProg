# 🎯 快速修复：蓝图摄像机输入禁用

## 问题原因

您的摄像机控制可能在**蓝图**中，C++无法拦截蓝图事件。

---

## ⚡ 3步快速修复

### 1️⃣ 打开角色蓝图

找到 `BP_MyCharacter` 或您的角色蓝图

### 2️⃣ 修改 Turn 事件

在 Event Graph 中找到 `InputAxis Turn`：

**原来**:
```
InputAxis Turn → Add Controller Yaw Input
```

**改成**:
```
InputAxis Turn → Should Block Camera Input? → Branch
                                                ↓ False
                                            Add Controller Yaw Input
```

### 3️⃣ 修改 LookUp 事件

在 Event Graph 中找到 `InputAxis LookUp`：

**原来**:
```
InputAxis LookUp → Add Controller Pitch Input
```

**改成**:
```
InputAxis LookUp → Should Block Camera Input? → Branch
                                                  ↓ False
                                              Add Controller Pitch Input
```

## 🔍 如何找到输入事件

### 方法 1: 搜索
1. 在 Event Graph 中按 `Ctrl+F`
2. 搜索 "Turn" 或 "LookUp"

### 方法 2: 查看所有事件
1. 在 Event Graph 的空白处右键
2. 选择 `Add Event` → `Input` → `Axis Events`
3. 查看是否有 `Turn`、`LookUp`、`TurnRate`、`LookUpRate`

## ⚠️ 重要提示

### 如果找不到蓝图输入事件

可能有3种情况：

#### 情况 1: 在父类蓝图中
- 打开父类蓝图检查
- 在子类蓝图中覆盖事件

#### 情况 2: 完全使用C++
- 蓝图中没有输入事件
- **但问题依然存在** → 请检查：
  1. 是否重新编译了代码
  2. 是否重启了编辑器
  3. 查看下方"高级诊断"

#### 情况 3: 使用增强输入系统
- UE5 的 Enhanced Input
- 需要不同的配置方式（见"增强输入系统配置"部分）

## 🛠️ 高级诊断

### 检查1: 确认C++代码生效

在游戏运行时查看输出日志，应该看到：

```
LogTemp: Warning: [INPUT LOCK] Camera control DISABLED
```

如果看到这个日志，说明C++代码已经执行，但蓝图绕过了C++。

### 检查2: 临时禁用蓝图输入

1. 在角色蓝图中找到所有 `InputAxis` 事件
2. 临时**禁用**这些事件（右键 → Disable）
3. 测试摄像机是否能旋转
   - **能旋转** → C++绑定有效，问题在蓝图
   - **不能旋转** → 摄像机控制在其他地方

### 检查3: 查看 PlayerController

有时摄像机控制在 PlayerController 中：

1. 找到您的 PlayerController 蓝图
2. 检查是否有 `Turn`、`LookUp` 输入事件
3. 如果有，应用相同的修复方法

## 🆘 仍然无效？

### 选项1: 发送截图
请截图以下内容：
1. 角色蓝图的 Event Graph（显示输入事件）
2. 游戏运行时的输出日志
3. Project Settings → Input → Axis Mappings

### 选项2: 使用纯C++方案

修改头文件添加：

```cpp
UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Camera")
bool bUseBlueprintCameraControl = false;
```

然后：
- `bUseBlueprintCameraControl = False` → 删除所有蓝图输入事件
- `bUseBlueprintCameraControl = True` → 在蓝图中添加锁定检查

### 选项3: 替代方案 - 禁用Controller Rotation

如果实在无法拦截输入，可以改用这个方法：

在 `StartLockOn()` 中：
```cpp
// 禁用控制器旋转
bUseControllerRotationYaw = true;
bUseControllerRotationPitch = true;
```

在 `CancelLockOn()` 中：
```cpp
// 恢复控制器旋转
bUseControllerRotationYaw = false;
bUseControllerRotationPitch = false;
```

**注意**：这个方法会改变角色旋转行为，可能需要调整其他系统。

---

## ✅ 总结

**核心修改**: 在蓝图的 `InputAxis Turn` 和 `InputAxis LookUp` 事件中，添加 `Should Block Camera Input?` 检查，只有在返回 `False` 时才执行摄像机旋转。

**原理**: C++的 `bIsLockedOn` 通过 `ShouldBlockCameraInput()` 函数暴露给蓝图，蓝图查询这个状态来决定是否处理输入。

**时间**: 整个修改应该在5分钟内完成 ⏱️
