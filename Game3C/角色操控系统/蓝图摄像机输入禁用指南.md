# 🎮 蓝图摄像机输入禁用设置指南

## 📋 问题诊断

### 为什么C++拦截无效？

**原因**：您的角色蓝图中可能有直接的输入绑定，例如：
- `InputAxis Turn` → `Add Controller Yaw Input`
- `InputAxis LookUp` → `Add Controller Pitch Input`

这些蓝图事件会**绕过C++的函数**，直接调用引擎的输入处理，所以C++的 `if (bIsLockedOn) return;` 无法拦截。

---

## ✅ 解决方案

在您的**角色蓝图**（BP_MyCharacter）中修改输入事件，添加锁定状态检查。

### 步骤 1: 打开角色蓝图

1. 打开您的角色蓝图（通常是 `BP_MyCharacter` 或类似名称）
2. 切换到 **Event Graph** 标签页

---

### 步骤 2: 修改 Turn 输入事件

找到 `InputAxis Turn` 事件节点（如果存在），修改如下：

#### ❌ 原始蓝图（错误）

```
[InputAxis Turn] → [Add Controller Yaw Input]
```

#### ✅ 修改后的蓝图（正确）

```
[InputAxis Turn]
    ↓
[Should Block Camera Input?] (调用C++函数)
    ↓ (False Branch)
[Add Controller Yaw Input]
```

#### 详细节点连接：

1. **InputAxis Turn** (Axis Value 输出)
   - 拖出 `Axis Value` → 右键搜索 **"Should Block Camera Input"**
   
2. **Should Block Camera Input** (Branch节点)
   - `Return Value` → 右键搜索 **"Branch"**
   
3. **Branch** 节点
   - `Condition` ← 连接 `Should Block Camera Input` 的返回值
   - `False` 分支 → **Add Controller Yaw Input**
   - `Axis Value` → 连接到 `Add Controller Yaw Input` 的 `Val` 输入
   - `True` 分支 → **不连接**（锁定时不执行）

---

### 步骤 3: 修改 LookUp 输入事件

找到 `InputAxis LookUp` 事件节点（如果存在），应用相同的逻辑：

#### ✅ 修改后的蓝图

```
[InputAxis LookUp]
    ↓
[Should Block Camera Input?]
    ↓ (False Branch)
[Add Controller Pitch Input]
```

#### 详细节点连接：

1. **InputAxis LookUp** (Axis Value 输出)
   - 拖出 `Axis Value` → 右键搜索 **"Should Block Camera Input"**
   
2. **Should Block Camera Input** (Branch节点)
   - `Return Value` → 右键搜索 **"Branch"**
   
3. **Branch** 节点
   - `Condition` ← 连接 `Should Block Camera Input` 的返回值
   - `False` 分支 → **Add Controller Pitch Input**
   - `Axis Value` → 连接到 `Add Controller Pitch Input` 的 `Val` 输入
   - `True` 分支 → **不连接**（锁定时不执行）

## 🛠️ 可选方案：完全使用C++

如果您想完全使用C++管理输入，可以：

### 方案 A：删除蓝图中的输入事件

1. 在角色蓝图的 Event Graph 中
2. 删除所有 `InputAxis Turn`、`InputAxis LookUp` 等输入事件
3. C++的 `SetupPlayerInputComponent` 会自动接管

### 方案 B：添加配置参数（推荐）

在头文件中添加：

```cpp
UPROPERTY(EditAnywhere, BlueprintReadWrite, Category = "Camera")
bool bUseC ppCameraControl = false;
```

然后在蓝图中：
1. 如果 `Use Cpp Camera Control` = **True** → 删除蓝图输入事件
2. 如果 `Use Cpp Camera Control` = **False** → 保留蓝图事件，但添加锁定检查

## 📝 常见问题

### Q: 我找不到蓝图中的输入事件？

**A**: 可能有两种情况：
1. **输入在父类蓝图中** - 打开父类蓝图检查
2. **使用C++绑定** - 检查C++的 `SetupPlayerInputComponent` 是否已经绑定

### Q: 蓝图输入和C++输入会冲突吗？

**A**: 不会冲突，但会**叠加**：
- 如果蓝图和C++都绑定了 `Turn` 轴
- 两者都会被调用
- 所以两边都需要添加锁定检查

### Q: 我应该用蓝图还是C++？

**A**: 推荐方案：
- **设计师友好**: 使用蓝图 + C++检查函数
- **程序员友好**: 纯C++，删除蓝图输入事件
- **灵活方案**: 添加配置参数，支持两种模式

## 🎯 总结

### 核心原理

C++的 `bIsLockedOn` 状态是**数据源**，蓝图通过 `ShouldBlockCameraInput()` 函数**查询**这个状态来决定是否处理输入。

### 工作流程

```
输入事件 
    ↓
蓝图查询 C++ 的 bIsLockedOn
    ↓
如果未锁定 → 处理输入
如果锁定   → 忽略输入
```

### 关键函数

```cpp
// 在头文件中已添加
UFUNCTION(BlueprintPure, Category = "Lock-On")
bool ShouldBlockCameraInput() const { return bIsLockedOn; }
```

这个函数可以在蓝图的任何地方调用，返回 `true` 表示应该阻止输入。
