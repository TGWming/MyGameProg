## 代码层面怎么做

```
玩家按键
  ↓
代码判断：玩家想做什么？
  ├─ 普通攻击 → 检查 bCanCombo（普通连段窗口）→ 开了就执行
  └─ 变形斩   → 检查 bCanTransform（变形斩窗口）→ 开了就执行
```

就是**多加一个窗口 Bool**：

| Notify | 控制的变量 | 打开时机（举例） |
|---|---|---|
| `ANS_ComboWindow` | `bCanCombo = true/false` | 0.5 ~ 0.75 |
| `ANS_TransformWindow` | `bCanTransform = true/false` | 0.3 ~ 0.75 |

每种"优先级不同的动作"就对应一个窗口。窗口放在 Montage 时间轴的哪个位置，完全由你决定。

## 推广到更多情况

| 窗口类型 | 什么动作能在这个窗口执行 | 打开时机 | 为什么更早/更晚 |
|---|---|---|---|
| `ANS_ComboWindow` | 同类型普通连段 | 较晚（0.5） | 必须让当前招式打出实质效果后才能接 |
| `ANS_TransformWindow` | 变形斩 | 较早（0.3） | 血源的变形斩设计上就是可以更快取消 |
| `ANS_DodgeCancelWindow` | 翻滚取消 | 更早（0.25） | 魂like 中翻滚的取消优先级通常最高 |
| `ANS_SkillCancelWindow` | 战技取消 | 视设计而定 | 有些战技可以很早打断普通攻击 |

所有这些都是**在同一个 Montage 上多放几条 NotifyState**，不同的 Notify 在不同的时间点打开不同的 Bool。

```
一个 Montage 的时间轴上可以同时叠很多 Notify：

0.0                                              1.0
 |                                                 |
 ├── ANS_DodgeCancelWindow ═══════════════════╗    |
 |                                            ║    |
 ├────── ANS_TransformWindow ═══════════╗     ║    |
 |                                      ║     ║    |
 ├────────── ANS_ComboWindow ═════╗     ║     ║    |
 |                                ║     ║     ║    |
 ├── AN_EnableHitbox              ║     ║     ║    |
 ├────── AN_DisableHitbox         ║     ║     ║    |
 |                                ╚═════╩═════╝    |
 ├─────────────────── AN_ActionEnd                 |
 |                                                 |

翻滚取消：最早开，最晚关（优先级最高）
变形斩：  比连段早开
普通连段：最晚开（优先级最低）
```

## 修正 1：BlendSpace 的规划

你说的：
> 前后移动快速+慢速，左右移动快速+慢速，Sprint

实际上不需要这么多独立的 BlendSpace。

### 非锁定移动（FreeMove）

只需要**一个 2D BlendSpace**：

```
BS_Locomotion_FreeMove_2D

横轴：Direction（-180 到 180）
纵轴：Speed（0 到 600）

里面放的动画点位：

        Speed
         600  ┌─ Run_F ──── Run_FR ──── Run_R ──── Run_BR ──── Run_B ┐
              │                                                        │
         300  ├─ Walk_F ─── Walk_FR ─── Walk_R ─── Walk_BR ─── Walk_B ┤
              │                                                        │
           0  └─ Idle ──── Idle ──── Idle ──── Idle ──── Idle ────────┘
             -180   -135    -90     -45      0      45     90    135   180
                                    Direction
```

玩家推摇杆 → 代码算出 Speed 和 Direction → BlendSpace 自动混合出对应的走/跑/方向。

**不需要分开做"前后快速""前后慢速""左右快速""左右慢速"。** 一个 2D BlendSpace 全包了。

### 锁定移动（LockOnStrafe）

需要**另一个 2D BlendSpace**：

```
BS_Locomotion_LockOn_2D

横轴：RightAxis（-1 到 1）
纵轴：ForwardAxis（-1 到 1）

里面放的动画点位：

       Forward
         1   ┌─ Strafe_FL ── Strafe_F ── Strafe_FR ┐
             │                                       │
         0   ├─ Strafe_L ─── Idle ──── Strafe_R ────┤
             │                                       │
        -1   └─ Strafe_BL ── Strafe_B ── Strafe_BR ─┘
            -1              0              1
                          Right
```

角色面朝敌人不转身，八方向平移。

### Sprint

Sprint 不需要 BlendSpace，**一个单独的 Animation Sequence 就够**。

因为冲刺时：
- 速度是固定的最高速（不需要混合不同速度）
- 方向永远是正前方（不需要混合方向）

```
所以 Locomotion 总共需要：

├── BS_Locomotion_FreeMove_2D    ← 1个 BlendSpace（非锁定，包含走/跑/全方向）
├── BS_Locomotion_LockOn_2D     ← 1个 BlendSpace（锁定，八方向平移）
├── AS_Sprint_Forward            ← 1个动画（冲刺）
└── AS_Idle                      ← 1个动画（站着）
    AS_Idle_Combat               ← 1个动画（战斗站姿，可选）

就这些。不需要更多了。
```

## 修正 2：你说不做负重变换所以不需要两套 BS

**你的判断是对的。** 如果不做负重影响动画姿态（比如黑魂3里负重高翻滚变胖翻），那确实不需要两套 Locomotion BS。

但你**需要两套 BS 的原因不是负重，而是锁定和非锁定**：

| BS | 什么时候用 | 为什么需要 |
|---|---|---|
| `BS_Locomotion_FreeMove_2D` | 没锁定敌人时 | 角色朝移动方向转身，用 Speed + Direction |
| `BS_Locomotion_LockOn_2D` | 锁定敌人时 | 角色面朝敌人不转身，八方向平移，用 Forward + Right |

这两个 BS 里的动画**完全不同**（转身跑 vs 面朝前方侧移），所以必须是两个。
